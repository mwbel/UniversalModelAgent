# 第五章 分支程序设计

在解决实际问题的过程中，处理进行到某一步骤后，经常会因具体条件的需要把下面的处理分成两支来进行。例如，让小学生做一道数学题，得到的答案可能是对的，也可能是错误的。如果答对了，可以再出更难些的题目让他去练习；如果错了，就要给他讲解。一般说来，用计算机处理这类问题，需要执行一种特殊结构的程序，即这一章要介绍的分支结构的程序。

# 第一节分支结构的程序

# 一、条件语句

例5.1里程序中的60语句行叫做条件语句。条件语句的格式是$\textbf { I F } <$ 关系式 $>$ THEN $<$ 语句组 $>$

像例5.1中的 $\displaystyle \mathbf { S } = \mathbf { X }$ 这样表示条件的式子，叫做关系式。关系式由用关系运算符连接起来的表达式组成。BASIC语言中的关系运算符与数学中常用的略有不同，一般用 $> =$ 表示“大于或等于”，用 $< =$ 表示“小于或等于”，用 $< >$ 表示“不等于”。其他关系运算符与数学符号相同，即用 $=$ 表示“相等”，用 $>$ 表示“大于”，用 $<$ 表示“小于”。

计算机执行条件语句时，先判断关系式给出的条件是否成立，如果成立就执行THEN后面的语句组 (THEN后面可以有用冒号隔开的多个语句)；如果条件不成立，就执行这个条件语句下面的语句行，而不执行THEN后面的语句。这样就把处理过程依条件是否成立而分成了两支。

英语单词IF有“如果”的意思，THEN有“那么就”的意思。从字面上看，60语句行的意思是：“如果 $\mathbf { S } = \mathbf { X }$ ,那么就输出‘对了，非常好！’，然后结束”，程序运行时也确实如此。

例5.1中60语句行的最后是END语句，所以程序进人这个分支，显示出信息后就结束了。如果THEN后面没有END语句，情况又会怎样呢？请看下面的例题。

例5.2某单位每月向各用户收煤气费，计费办法如下：4人以下和4人的用户，每户收6元；超过4人的用户，每超出1人加收1.2元。设计一个程序，按输人的人数计算应收费的金额。

程序：

20 INPUT"人数 $\mathbf { \Lambda } = \mathbf { \Lambda } ^ { \prime \prime } ; \mathbf { N }$   
30IF $\mathbf { N } { > } 4$ THEN LET $\mathbf { Y } = \mathbf { Y } + \left( \mathbf { N } - 4 \right) * 1 . 2$   
40 PRINT“应收 ${ \pmb \mathscr { n } }$ ；Y；"元"   
50 END

运行这个程序，执行到语句20,屏幕上显示出“人数 $=$ ”时，先输入一个小于4的人数，例如 3,屏幕上会显示出

应收6元 1

再运行一次，输入一个大于4的人数，例如7,这时屏幕上会显示出

]

两次的结果都是正确的。用不同的值输入时都能输出正确的结果，这说明当输入的数大于4时，语句30中的关系式 $\mathbf { N } { > } 4$ 成立，计算机执行THEN后面的赋值语句，把Y的值增加了应增的金额，然后也执行语句40,输出了处理的结果。

从这个例子可以看到，当条件语句的THEN后面没有END语句时，如果条件成立，计算机执行完THEN后面的语句序列后，继续按行号的顺序向下执行。

# 二、分支结构的程序

用条件语句编程序解决实际问题的过程中，依照一定的条件分别处理时，每一个分支可能有较多的步骤。这时，需要使用GOTO语句构成完整的分支结构。请看下面的例子。

例5.3某公司批发软磁盘。购货数量不足500片时，按定价每片4.5元收款，购货数量达到或超过500片时，按定价的 $8 5 \%$ 收款。据此，可以编出下面这样的程序。

10 HOME  
20 INPUT“请输入购货数量："；N  
30 IF $\mathbf { N } { < } 5 0 0$ THEN LET ${ \bf P } { = } 4 . 5$ : GOTO 50  
40 LET $\mathbf { P } { = } 4 . 5 { * } 8 5 / 1 0 0$   
50 LET ${ \mathrm { T L } } { = } \mathbf { P } { * } \mathbf { N } $   
60 PRINT  
70 PRINT"数量",”单价",”金额"  
80 PRINT N,P, TL  
90 END

程序中30语句行是一个条件语句。THEN后面是用冒号隔开的两个语句，后一个叫做GOTO语句。GOTO语句的语句定义符取自英语go to,意为“到……去”。语句定义符后面是程序中的某个语句行的行号。从字面上看，GOTO语句的意思是：“到xx行去继续执行”。

GOTO语句的格式是

GOTO $<$ 行号 $>$

有了GOTO语句的控制，运行这个程序输人了小于500的数时，计算机会先执行30语句行中 THEN后面的LET语句，把表示单价的变量P的值变成4.5,然后执行后面的GOTO50,转到语句行50去按照每片4.5元的单价计算货款金额。然后再顺序执行60、70、80等语句行。

# 二、逻辑判断

在分支程序里，利用条件语句对关系式的判断，可以处理一些较简单的问题。实际解决问题时，给出的条件往往比较复杂，需要先进行逻辑运算才能确定条件是否成立。请先看一个例子。

在这个程序里，40语句中的关系式之间使用了AND、OR等符号。在BASIC语言里，这类符号叫做逻辑运算符。用逻辑运算符把关系式连接起来构成的式子叫做逻辑表达式，简称逻辑式。逻辑式表示的是逻辑运算。有关逻辑运算的知识，我们将在第十一章里讲解。在这里只介绍一些最基本的内容。

# 1.三种逻辑运算

例 5.6中用到的AND和OR 这两个逻辑运算符，分别表示“与”运算和“或”运算。

(1）“与”运算用AND把关系式连接起来表示的逻辑运算叫做“与”运算。例如，$\mathrm { C H } > = 9 0 $ AND $\mathbf { M A } > = 9 0$ 表示的就是两个关系式的“与”运算。英文单词AND在这里有“并且”的意思。因此， $\mathrm { C H } > = 9 0 $ AND $\mathbf { M A } > = 9 0$ 这个逻辑式的意思是：“CH的值不小于90并且MA的值也不小于 $9 0 ^ { \eta }$ 。“与”运算的意义是：只有参加运算的所有关系式都成立时，整个运算的逻辑式才成立。

(2）“或”运算用OR把关系式连接起来表示的逻辑运算叫做“或”运算。例如， $\mathbf { A } > = 9 5$ OR $\mathbf { B } > = 8 5$ 表示的就是两个关系式的“或”运算。英文单词OR在这里有“或者”的意思。因此， $\mathbf { A } > = 9 5$ OR $\mathbf { B } > = 8 5$ 这个逻辑式的意思是：“A的值不小于95,或者B的值不小于 $8 5 "$ 。“或”运算的意义是：只要参加运算的逻辑式中有一个成立，整个“或”运算的逻辑式就成立。

(3）“非”运算除了AND和OR以外，常用的还有一种“非”运算。“非”运算用英文单词NOT表示。例如，NOT $\mathbf { M A } > = 8 0$ 表示对 $\mathbf { M A } > = 8 0$ 这个关系式进行的“非”运算。NOT表示否定，相当于“不是”。逻辑式NOT $\mathbf { M A } > = 8 0$ 的意思是：“不是MA的值不小于 $8 0 ^ { \circ }$ ，也就是“MA的值小于 $8 0 ^ { \circ }$ 。“非”运算的意义是：参加运算的关系式如果成立，整个“非”运算的逻辑式就不成立；反之，参加运算的关系式如果不成立，整个“非”运算的逻辑式就成立。

逻辑运算的结果还可以再参加逻辑运算。例如， $\mathrm { N O T } ( \mathrm { A } < = 5 $ AND $\mathbf { A } { > } 0$ OR $\mathbf { A } { > } 1 0 0 )$ 以及这个例子里的 $\mathrm { C H } > = 9 0 $ AND $\mathbf { M A } > = 9 0$ OR $\mathbf { C H } + \mathbf { M A } > = 1 8 5$ 等。运算中，执行三种逻辑运算的优先顺序是，先执行“非”运算，其次执行“与”运算，最后执行“或”运算。如果使用了括号，优先进行括号里的运算。

# 2.逻辑值

在BASIC语言里，关系式也叫逻辑式。逻辑式的成立与否，是根据它的逻辑值来确定的。逻辑式只有“成立”和“不成立”两种可能。相应地，逻辑值只有真（true)和假(false)两个。在BASIC语言里，一般用1表示真，用O表示假。用PRINT命令或 PRINT语句都可以输出逻辑式的值。例如，用PRINT $5 { < } 9$ 可以输出1（表示 $5 { < } 9$ 成立)，用PRINT $3 + 4 = 1 2$ 可以输出0（表示 $3 + 4 = 1 2$ 不成立)，等等。

# 本章小结

1．本章介绍了分支程序。分支程序的核心是条件语句。条件语句也叫IF-THEN语句。计算机执行条件语句时，先判断表示条件的关系式是否成立，如果成立就执行THEN后面的语句组，使处理进入一个分支，如果不成立就执行条件语句下面的那个语句行，使处理进入另一个分支；在分支程序里，由条件语句控制计算机选择两个分支之一进行处理，无论选择了哪个分支，处理结束后都要回到程序的主干上来。也就是说，分支程序只有一个入口和一个出口。

2．编写分支程序时，一般应先设计IF-THEN语句下面条件不满足时的那一个分支，然后再设计条件满足时THEN后面的那一个分支。需要时，注意用GOTO语句使分支程序有唯一的出口。

3．用关系式 $\mathbf { X } / \mathbf { Y } { = } \mathbf { I N T } ( \mathbf { X } / \mathbf { Y } )$ 是否成立，可以判断X是否能被Y整除。

# 第一节用条件语句构成循环

利用 FOR－NEXT语句可以控制循环体的执行次数，但在实际应用中，有些问题无法事先确定循环的次数，在这种情况下，可以适当地应用条件语句来处理。请看下面的例子。

例6.1 在菲波那契数列1,1,2,3,5,8,13,21,中，从第三项开始，每一项等于它前面两项的和。数列中的第几项的值刚好超过或等于10000?

分析：构造一个循环程序，每执行一次循环体生成菲波那契数列的一项，用这项的值与10000 比较，如果不小于10000,就输出结果，否则返回去再生成下一项，再比较，……，直到生成了不小于10000的那一项为止，这样就可以使问题得到解决。需要注意的是每生成一项，要记下项数。据此，可画出图6.1所示的框图。

程序：

10 LET $\scriptstyle Q = 1$ ：LET $\mathbf { S } = 1$   
20 LET $\mathbf { N } = 2$   
30 LET ${ \bf N } = { \bf N } + 1$   
40 LET $\mathbf { F } { = } \mathbf { Q } { + } \mathbf { S }$   
50IF $\mathrm { F } { < } 1 0 0 0 0$ THEN LET $\scriptstyle { Q = \mathbf { S } }$ : LET $\mathbf { S } = \mathbf { F }$ : GOTO 30   
60 PRINT " $\mathbf { N } { } = { } ^ { \prime }$ ; N,"F=";F   
70 END

运行这个程序，屏幕上会显示出

$$
\mathsf { F } = 1 0 9 4 6
$$

表示这个数列的第 21项刚好超过10000,具体的值是10946。

结合图6.1可以清楚地看到，程序中 $3 0 \sim 5 0$ 语句行构成了一个循环，循环的次数由条件语句中关系式 $\mathrm { F } { < } 1 0 0 0 0$ 是否成立来控制。如果关系式成立，就反复执行有关的语句，生成新的数列项并记下项数；如果关系式不成立，就结束循环，输出结果。

# 第二节循环的嵌套

循环程序中间还可以包含循环程序。请看下面的例子。

例6.3 双重循环程序的执行过程。

10 LET $\scriptstyle \mathrm { L } 1 = 0 \ : \ \mathrm { L E T } \ \scriptstyle \mathrm { L } 2 = 0$   
20 FOR $\mathbf { I } = 1$ TO 3   
30 LET $_ { \mathrm { L 1 = L 1 + 1 } }$   
40 FOR $\mathbf { J } = 1$ TO 10   
50 LET $_ { \mathrm { L } 2 = \mathrm { L } 2 + 1 }$   
60 PRINT " \*" ;   
70 NEXT J   
80 PRINT   
90 NEXT I   
100 PRINT $" \mathrm { \bf ~ L } 1 = "$ ; L1, $" \mathrm { ~ L } 2 = "$ ; L2   
110 END

运行这个程序，屏幕出会显示出

\*\*\*\*\*\*\*水\*\* \*\*\*\*\*\*\* ★\*\*\*\*\*\*\*\*\*

]

在这个程序里，语句20和语句90构成了外层循环，语句40和语句70构成了内层循环。运行时，由20,90这两个语句控制着计算机执行 $3 0 \sim 8 0$ 语句构成的外循环体，每执行一次给计数变量L1加1。执行外循环体时，又由40,70这两个语句控制计算机执行语句50和语句60构成的内循环体。语句 50给另一个计数变量加1,语句 60输出1个 $^ *$ 号。从运行结果可以看到，外循环体被执行了3次，内循环体被执行了 $3 \times 1 0 = 3 0$ 次。语句80的作用是每输出10个 $^ *$ 号后，使下次输出的内容换行。

在程序设计中，这种循环套循环的程序结构，叫做循环嵌套，也叫多重循环。BASIC语法规定，循环嵌套时不能出现交叉。例如，图6.3(a)表示正确的循环嵌套，图6.3(b)表示不正确的嵌套。嵌套不正确时，计算机不能正常运行。

# 第三节检索与排序

利用计算机的判断功能和它运行时的高速度，可以从大量的数据中很快找到满足人们不同需要的数据，这就是检索。为了满足分析、研究的需要，按照一定的规则为大量数据排出顺序就是所谓的排序。在进行这类处理时，计算机一般要先把数据存放在内存储器里一些事先准备好的存储单元里，用特定的变量来指明数据存放的位置。这类变量就是本节要介绍的下标变量。

# 一、单下标变量与一维数组

用BASIC语言编程序时，是通过变量来进行对内存储器的存取操作的。例如，用LET$\mathbf { A } = 1 0 0$ 把100这个数据存入内存储器中变量A指定的存储单元里，用PRINTX,Y从内存储器中变量X和Y指定的存储单元里取出数据并显示在屏幕上。到目前为止，我们使用过的变量，如A,B,X,Y,F1,F2等,叫做简单变量。简单变量不能反映出数据存储的顺序，即使是像F1,F2这样带有数字的简单变量，对于计算机来说也只是两个不同的变量而已。能反映出数据存储顺序的变量，是下标变量。下标变量由下标变量名和由括号括起来的下标构成，形如A(n)。其中A是下标变量名，下标变量的命名规则与简单变量相同；n是下标变量的下标，下标必须放在圆括号里,可以是数、简单变量或其他算术表达式，也可以是下标变量，如A(1),B(2),X(M),Y(A()),$\mathbf { B } ( \mathbf { X } + 1 )$ 等等。下标表示的是下标变量的序号，因此下标的值只能是0或正整数。如果下标值是小数，计算机会自动取整。

具有相同名称的下标变量的集合叫做数组。数组用来存放一组相互关联的数据。数组用组成它的那些下标变量的名称来命名。例如，由 $\mathrm { H } ( 0 )$ ，H(1),，H(9)等下标变量组成的数组，叫做“数组 $\mathbf { H } ^ { \ast }$ 。属于某一个数组的各下标变量叫做该数组的元素。

存放字符串的数组叫字符串型数组，字符串型数组的元素是字符串型的下标变量。字符串型下标变量名的末尾要有一个 $\$ 8$ ，如 $\mathbf { A } \mathbb { S } ( 0 )$ ， $\mathbf { B } \mathbb { S } ( \mathbf { X } - 1 )$ ， $\mathbf { X } { \mathfrak { S } } ( \mathbf { A } ( 9 ) ) ,$ 等等。

像 A(1), $\mathbf { B } \mathbb { S } ( \mathbf { X } - 1 )$ 这样只有一个下标的下标变量，叫做单下标变量；由单下标变量组成的数组，叫做一维数组。

# 本章小结

1．解决实际问题时，如果无法预先知道循环次数，可以用条件语句或条件语句配合GOTO语句构成循环，否则应使用FOR-NEXT语句来构造循环程序。为了使程序条理清楚，应尽量限制GOTO语句的使用。

2．用多重循环处理问题时，要注意循环嵌套的正确性。循环体中使用条件语句时，要正确控制程序转移的方向。

3．数组由下标变量组成。下标表明了该下标变量在数组中的位置。

4．检索数据时，一般先把数据存入一个数组，然后在数组中按规定的条件检查、搜索，从而找到所需的数据。

5．用二重循环为M个数据排序时，外循环变量的值应从1变到 $\mathbf { M } - 1 ,$ ，内循环变量的值每次应从外循环变量当前值加上1变到M。

# 第二节BASIC函数

# 一、标准函数

在数值计算中，经常需要使用数学函数，如对数函数、三角函数以及本书前面用过的求算术平方根函数 SQR(X)、取整函数 $\mathbf { I N T } ( \mathbf { X } )$ 、随机数函数RND(X)、绝对值函数ABS(X)等。在BASIC语言里，这些函数叫做标准函数或内部函数。

使用标准函数时，函数名必须用大写字母；自变量必须用圆括号括起来；自变量必须是符合规定的BASIC表达式；三角函数中自变量的单位必须是弧度。下面再请看几个应用其他标准函数解决问题的例子。

# 1．符号函数

BASIC 语言中有一个符号函数 SGN(X),当自变量 $\mathbf { X } { > } 0$ 时，函数值为1、 $\mathbf { X } { = } 0$ 时函数值为0、 $\mathbf { X } { < } 0$ 时函数值为－1。

# 三、自定义函数

除了标准函数和字符串函数以外，BASIC语言还允许程序设计者自己定义函数。请看下面的程序。

10 DEF FN ${ \bf A } ( { \bf X } ) { = } { \bf X } { * } { \bf X } { * } { \bf X }$

30 PRINT FN A(①),  
40 LET $\mathbf { S } = \mathbf { S } + \mathbf { F } \mathbf { N }$ A(1)

程序中的语句10叫做自定义函数语句。自定义函数语句的格式是

# DEF $<$ 函数名 $>$ （自变量） $= <$ 算术表达式 $>$

函数名必须以FN打头，后面跟一个可用于变量名的字符串，例如，在上面程序中的语句10里，用字母A与FN组成的函数名是FNA;圆括号中的自变量是一个形式参数，它只在自定义函数语句中起描述函数的作用，不会影响程序中变量的值，因此可以选用任何一个变量名；算术表达式的作用是告诉计算机如何进行计算，例如，语句10里用 $\mathbf { X } { * } \mathbf { X } { * } \mathbf { X }$ 告诉计算机要计算自变量的三次幂。

自定义函数语句的功能是定义一个函数，告诉计算机如何计算函数值，并给这个函数取一个名字。计算机执行自定义函数语句后，就可以像使用标准函数一样用函数名调用这个自定义函数了。

# 本章小结

1．处理较复杂的问题时，可以把相对独立或需多次使用的程序段写成子程序的形式，需要时由主程序调用；也可以把程序的各处理模块设计成子程序，把主程序设计成各处理模块的“调度”程序。

2．常用的标准函数有正弦函数SIN(X)、余弦函数 $\mathbf { C O X } ( \mathbf { X } )$ 、取整函数 $\mathbf { I N T } ( \mathbf { X } )$ 、随机数函数$\mathbf { R N D } ( \mathbf { X } )$ 、绝对值函数 $\mathbf { A B S } ( \mathbf { X } )$ 、符号函数 $\mathbf { S G N } ( \mathbf { X } )$ 、对数函数 $\mathbf { L O G } ( \mathbf { X } )$ 等；常用的字符串函数有LEN(XS)函数、LEFT\$（X\$,n） 函数、 $\mathbf { M D \ S } ( \mathbf { X \ S } , m \ , n )$ 函数、CHR\$(X)函数和 ASC(X\$)函数。

3．使用自定义函数之前，要先用自定义函数语句规定函数要进行的计算，自定义函数名必须以 FN 打头。

# 第八章 音乐与作图

# 第一节让计算机演奏音乐

# 一、发音单元

中华学习机和 AppleⅡ 型计算机里，有一个扬声器，利用它可以发出各种声音来。这两种计算机把内存储器的49200单元作为控制扬声器的开关。通过命令或语句访问这个存储单元时，主机的扬声器就会发出声音来。适当控制发音的频率，就可以让扬声器发出各种乐音。

10 FOR $\mathbf { I } { = } 7 7 0$ TO 790

20 READ A

50 DATA 173,48,192,136,208,5,206,1,3,240,9,202,208,245,174,0,3,76,2,3,96计算机执行这段程序后，通过READ语句和POKE语句，把一个产生乐音的机器语言子程序存入了内存储器的 $7 7 0 \sim ~ 7 9 0$ 单元。完成这个准备工作后，就可以运行音乐程序了。

# 第二节用计算机作图

计算机有较强的绘图功能，既可以用字符组成图案，又可以用色块或色点画出更为细致的图形来。

# 一、屏幕显示方式

中华学习机和AppleⅡ机的显示器有三种显示方式，即文本方式、低清晰度图形方式和高清晰度图形方式。开机后，显示器自动进入文本方式。这时，整个屏幕可以显示 24行 $\times 4 0$ 列$\mathtt { \mathtt { = 9 6 0 } }$ 个字符。在文本方式下，可以利用TAB(X)函数控制字符在屏幕上的输出位置，组成图案。但是，这种用字符组成的图案比较粗糙，实际绘图时，常常让显示器进人低清晰度图形方式或高清晰度图形方式。

在低清晰度图形方式下，屏幕上每行可以显示40个色块，整个屏幕最多可以显示 48行。用色块可以组成比较细致的图形。

在高清晰度图形方式下，屏幕上每行可以显示 280个色点，整个屏幕最多可以显示192行。用色点可以在屏幕上描绘出非常细致的图形来。

# \*第九章 程序设计方法

本书的第三章到第八章，介绍了用BASIC语言进行程序设计的基本知识。学习了那部分内容后，我们就可以用BASIC语言编写一些程序，让计算机执行它们，从而解决一些实际问题了。BASIC语言只是各式各样计算机语言中最容易学习的一种。用计算机解决实际问题时，还可以使用其他程序设计语言。不同的计算机语言的语法规则可能有很大的差异，但程序设计方法是独立于各式各样计算机语言之上的。掌握了程序设计方法的有关知识，就有了编程序应用计算机解决问题的基础，在此基础之上，熟悉某一种计算机语言后，就可以做到“触类旁通”了。

在介绍程序设计方法之前，我们先分析一下程序的基本结构。

# 第一节程序的基本结构

从前面几章中可以看到，程序是一个用一系列语句描述的用计算机解决问题的过程。由于要解决的实际问题的复杂程度各不相同，编出来的程序也就存在着或大或小、或简或繁的差异。不同的程序设计者为解决同一个问题所编的程序也会存在很大的差异。经过对大量程序的分析可以发现，尽管存在着各种差异，计算机在执行一个程序或一个程序段时，基本上只有三种情况：按语句的排列顺序执行、执行到某一语句时产生分支、重复执行一组语句。按照计算机执行情况的不同来分类，程序或程序段有三种不同的基本结构，即顺序结构、分支结构和重复结构。习惯上把构成整个程序的程序段或者组成一个程序系统的各种程序叫做模块。

# 二、结构化程序

顺序结构、分支结构、重复结构等程序的基本结构都只有一个人口和一个出口。各种基本结构模块可以通过复合与嵌套组成较大的模块。用较小的模块组成较大的模块时，较大的模块仍然只有一个出口和一个入口。如此继续下去，就可以构成整个程序。这种由基本结构模块逐步构造成的程序，就是所谓的结构化程序。可以认为，结构化程序是用基本结构模块通过复合、嵌套而构成的程序。

# 第二节结构化程序设计方法

编程序解决实际问题的过程大致可分三步来进行：分析问题，确定算法；实际编写程序；调试程序。实际编写程序是这一节要讨论的内容。首先看一下结构化程序设计的基本方法。

# 一、自顶向下，逐步求精

我们知道，编写程序是用计算机语言描述对实际问题的处理过程。这与写文章有很多相似的地方。写文章时，一般要先拟出提纲，把整个文章分成几大部分。然后，为每一部分再列出较细的提纲。这样逐步细分下去，直到拟出章、节、段落的细目。最后，按纲目充实以具体内容，经过加工、润色，一篇条理清晰、文字流畅的文章就脱稿了。

结构化程序设计的基本方法是“自顶向下，逐步求精”的方法。这种方法与写文章时采用的方法类似。用自顶向下逐步求精的方法进行程序设计时，先把对一个问题的处理过程粗分为几个模块；然后把每一个模块再细化为几个较小的模块；。这样一步步细化下去，直到每个小模块的处理功能直接用语句来实现，就完成了整个程序设计任务。

在逐步求精的过程中，每一步只是把一个模块细化为几个较小的模块，这种细化过程不容易出错，因而保障了整个程序的正确性。即便出现了错误，由于程序的结构清晰，便于由粗到细逐块查找与修改，而且，某一模块内的错误只需在本模块内进行适当的调整。

为了说明逐步求精的过程，请看一个简单的例子。

# 二、结构化程序设计方法的要点

结构化程序设计的基本方法是“自顶向下，逐步求精”。在具体实践中还要注意以下几个方面的问题。

$\textcircled{1}$ 把整个程序分成若干模块时，每个模块只能有一个人口和一个出口;

$\textcircled{2}$ 要按基本结构进行模块间的连接;

$\textcircled{3}$ GOTO语句只与IF语句配合使用，以构成分支或重复结构模块;

$\textcircled{4}$ 用必要的注释语句区分不同的模块并说明其功能;

$\textcircled{5}$ 把需多次使用的模块设计成子程序的形式;

$\textcircled{6}$ 程序编好后，如确实需要修改，要注意保持程序的模块化结构。

# 三、程序设计的准则

结构化程序设计为我们提供了良好的方法。要设计出结构、性能良好的程序，还需要注意程序设计的一些准则。

# 1.程序的正确性与可靠性

程序的价值首先在于能够运行并正确地解决问题，即在预定的条件下输入必要的数据后，能

得到正确的输出结果。其次，还要求程序必须稳定可靠，在遇到意外情况时能自动进行适当的处理。例如，程序能识别由于误操作输人的不合理的数据，并能采取适当的对策，以确保程序的运行不受干扰。可靠性指的是程序正常工作的能力，一般用平均无故障时间来衡量。

# 2.程序的可读性

所谓可读性，指的是对程序进行阅读与理解的方便程度。程序编好后，一般要经过多次修改才能确保其正确性与可靠性。要修改，就必须先读懂它、理解它。现代计算机的应用程序往往要若干人分工合作来完成。某个人编写的程序，不仅自己能读懂，还要使其他人也能够读懂，这样才便于整个程序系统的调试、组装等工作的顺利进行。为此，应尽可能地在程序中关键的位置上使用注释语句，说明程序模块的功能、变量的作用等问题。

# 3.程序的可维护性

程序运行时可能出现故障。出现故障后检查故障的原因、确定程序中的错误位置、修正程序中的错误等操作需要一定的时间。程序的可维护性的优劣就是以这类操作所需时间的长短来体现的。

# 4.程序的可移植性

程序的可移植性指的是在某一种型号的计算机上编写的程序，转到其他型号的计算机上去运行的能力。

# 5.程序的可扩充性

程序的可扩充性指的是在不改变整体结构的前提下，经过某些处理使程序的功能有所扩充。

# 6.程序的效率

程序的效率一般指程序运行时占有系统资源的多少与程序本身处理功能之间的比率。这里所说的系统资源，包括存储空间、机时等。如果一个不很长的程序在较短的时间内能够解决很复杂的问题，我们就说这个程序的效率比较高。反之，如果一个冗长的程序所能解决的只是比较简单的问题，那么它的效率就是很低的。由于现代计算机的存储容量很大、运算速度很高，为了保证程序的可靠性、可读性、可维护性、可扩充性等指标，在设计程序时可以把进一步提高程序效率的工作放在较次要地位上。

# 第三节程序的调试

编写工作结束以后，程序可能会有不完备的地方，也可能有错误。为确保程序能够顺利地完成预定的任务，还要对程序进行调试。在这里，调试是对程序进行检测、确定错误发生的部位、

排除错误等操作的总称。

# 一、程序中常见的错误

# 1．算法错误

在分析问题、确定算法时出现的失误，会造成算法错误。按照错误的算法编写出来的程序当然无法达到预定的目的。这类错误，计算机运行时检测不出来。要纠正算法错误，必须重新分析问题、确定正确的算法，重新设计程序。

# 2．语法错误

各种程序设计语言都有严格的语法规则。初学者不熟悉这些语法规则或者在输入程序时的失误等，都会造成程序中出现语法错误，使程序不能正常运行。

运行 BASIC程序时，若程序中有语法错误，计算机会在执行到有错误的语句时停下来，并显示出一些信息指明错误的类型与出错语句的行号。

# 3.不违反语法规则的录入错误

如果输入的数据有错误，例如输入小数点时按了逗号键、输入变量名时按错了字母键等。这类错误没有破坏语句的语法结构，程序还会运行下去，但往往会得到错误的结果。计算机不能自动检测出这类错误。

# 4.逻辑错误

由于设计不当，程序中可能会隐含着逻辑错误。某些逻辑错误可以中断程序的运行，例如，进行除法运算时除数为零、试图对负数进行开平方运算等。但多数逻辑错误不违背语法规则，因此，计算机不能自动检测出来。例如，由于计算机计算时产生的误差，使得 $\mathbf { X } { = } 9$ 时，关系式$\mathbf { X } { \hat { \mathbf { \theta } } } 2 { = } 8 1$ 并不成立，导致在程序中用 IF $\mathbf { X } { \hat { \mathbf { \theta } } } 2 = 8 1 $ THEN…这样的语句行处理问题时，永远不会执行THEN后面的语句。查找这样的逻辑错误往往比较困难。

# 二、程序的调试方法

# 1．自底向上逐层调试

我们知道，结构化程序设计的基本方法是自顶向下逐步求精。程序的调试过程应反过来，自底向上逐层调试，即先调试最低层次的模块，确实无误后，再调试由这些最低层次模块构造成的上一个层次的模块。这样逐层调试，直到整个程序完成调试操作。对各模块进行的调试叫做分调。大型软件的分调可以用多台计算机分头进行。分调完成后，要把所有的模块装配起来进行联调。所谓联调就是分调后对整个程序进行的调试。具体调试时，可以采用下面介绍的一些技巧与方法。

# 2.利用BASIC解释程序自动检测语法错误的功能

程序运行中断，屏幕上显示出出错信息时，可以根据出错信息的提示调出有语法错误的语句，分析一下产生错误的原因，修改有关的语句后再继续运行。

# 3.输入适当的初始数据进行程序的试运行

编好程序后，可以先提供一些初始数据让程序进行试运行。所提供的初始数据要有代表性，运行结果应是可预知的，以便利用试运行输出的结果来分析程序的正确性。例如，要调试解一元二次方程 $a x ^ { 2 } + b x + c = 0$ 的程序，可以把2、9、4作为方程的系数输入，看看输出的两个根是否为 $- 0 . 5$ 和-4;把2、4、2作为方程的系数输入，看看输出的两个根是否都是-1;把7、2、5作为方程的系数输入，看看输出的结果是否表示无实根。若发现错误，可以到相应的模块中去查找。需要说明的是，利用这种方法往往可以找出程序中的错误，但不能保证程序没有错误，所以还要配合其他方法来调试。

# 4.按 $\mathbf { C } \mathbf { m } + \mathbf { C }$ 键中断程序的运行

发现程序可能有逻辑错误时，如运行时间超长、不能正常结束等，可以按 $\mathrm { C t r l } { + } \mathrm { C }$ 键使程序停止运行。然后用立即执行方式的命令，如PRINT命令，检查有关变量的值。通过对有关变量当前值的分析，可以找出错误的根源。例如，程序运行到一个循环内部不能结束时，可以用上述方法检查循环控制变量的当前值，了解一下循环体执行了多少次，然后进一步确定出现错误的原因。

中断程序的运行以后，经过检查可以用立即执行方式的赋值语句改变某些变量的值，然后用CONT命令使程序再继续运行，进一步观察程序的运行情况。如果修改了程序中的语句，必须用RUN命令使程序的运行从头开始。

# 5.设置断点

在程序中插入一些STOP语句，可以使计算机遇到STOP语句时中断程序的运行。插入这些STOP 语句，相当于在程序中设置了一些断点。用设置断点的方法，可以对程序分段调试。如未发现错误，用CONT命令可以使程序继续运行。

断点的位置应设在程序中各个模块的末尾，以便有计划地逐块检查程序的运行状态。调试完毕，应删除那些STOP语句，把断点取消。

# 6.跟踪程序的运行

调试程序时，可以临时在程序中某些关键位置上插人一些PRINT语句，用来显示一些特定信息或某些变量值的动态状况，从而跟踪程序的运行。

程序调试完毕，要把插入的语句删除。
