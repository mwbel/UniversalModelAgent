# 第六章 较复杂的程序

前面三章介绍了顺序、循环和分支程序。综合使用这三种结构的程序设计方法，可以编出较复杂的程序，解决很多实际应用中问题。

# 第一节用条件语句构成循环

利用 FOR－NEXT语句可以控制循环体的执行次数，但在实际应用中，有些问题无法事先确定循环的次数，在这种情况下，可以适当地应用条件语句来处理。请看下面的例子。

例6.1 在菲波那契数列1,1,2,3,5,8,13,21,中，从第三项开始，每一项等于它前面两项的和。数列中的第几项的值刚好超过或等于10000?

分析：构造一个循环程序，每执行一次循环体生成菲波那契数列的一项，用这项的值与10000 比较，如果不小于10000,就输出结果，否则返回去再生成下一项，再比较，……，直到生成了不小于10000的那一项为止，这样就可以使问题得到解决。需要注意的是每生成一项，要记下项数。据此，可画出图6.1所示的框图。

程序：

10 LET $\scriptstyle Q = 1$ ：LET $\mathbf { S } = 1$   
20 LET $\mathbf { N } = 2$   
30 LET ${ \bf N } = { \bf N } + 1$   
40 LET $\mathbf { F } { = } \mathbf { Q } { + } \mathbf { S }$   
50IF $\mathrm { F } { < } 1 0 0 0 0$ THEN LET $\scriptstyle { Q = \mathbf { S } }$ : LET $\mathbf { S } = \mathbf { F }$ : GOTO 30   
60 PRINT " $\mathbf { N } { } = { } ^ { \prime }$ ; N,"F=";F   
70 END

运行这个程序，屏幕上会显示出

$$
\mathsf { F } = 1 0 9 4 6
$$

表示这个数列的第 21项刚好超过10000,具体的值是10946。

结合图6.1可以清楚地看到，程序中 $3 0 \sim 5 0$ 语句行构成了一个循环，循环的次数由条件语句中关系式 $\mathrm { F } { < } 1 0 0 0 0$ 是否成立来控制。如果关系式成立，就反复执行有关的语句，生成新的数列项并记下项数；如果关系式不成立，就结束循环，输出结果。

例6.2设计一个程序，从键盘上输入两个数，求出它们的最大公约数。

# 第二节循环的嵌套

循环程序中间还可以包含循环程序。请看下面的例子。

例6.3 双重循环程序的执行过程。

10 LET $\scriptstyle \mathrm { L } 1 = 0 \ : \ \mathrm { L E T } \ \scriptstyle \mathrm { L } 2 = 0$   
20 FOR $\mathbf { I } = 1$ TO 3   
30 LET $_ { \mathrm { L 1 = L 1 + 1 } }$   
40 FOR $\mathbf { J } = 1$ TO 10   
50 LET $_ { \mathrm { L } 2 = \mathrm { L } 2 + 1 }$   
60 PRINT " \*" ;   
70 NEXT J   
80 PRINT   
90 NEXT I   
100 PRINT $" \mathrm { \bf ~ L } 1 = "$ ; L1, $" \mathrm { ~ L } 2 = "$ ; L2   
110 END

运行这个程序，屏幕出会显示出

\*\*\*\*\*\*\*水\*\* \*\*\*\*\*\*\* ★\*\*\*\*\*\*\*\*\*

]

在这个程序里，语句20和语句90构成了外层循环，语句40和语句70构成了内层循环。运行时，由20,90这两个语句控制着计算机执行 $3 0 \sim 8 0$ 语句构成的外循环体，每执行一次给计数变量L1加1。执行外循环体时，又由40,70这两个语句控制计算机执行语句50和语句60构成的内循环体。语句 50给另一个计数变量加1,语句 60输出1个 $^ *$ 号。从运行结果可以看到，外循环体被执行了3次，内循环体被执行了 $3 \times 1 0 = 3 0$ 次。语句80的作用是每输出10个 $^ *$ 号后，使下次输出的内容换行。

在程序设计中，这种循环套循环的程序结构，叫做循环嵌套，也叫多重循环。BASIC语法规定，循环嵌套时不能出现交叉。例如，图6.3(a)表示正确的循环嵌套，图6.3(b)表示不正确的嵌套。嵌套不正确时，计算机不能正常运行。

例6.4 输出3到自然数M之间的素数。

分析：要解决这个问题，可以构造一个循环，让计算机产生3到M之间的所有奇数X。在这个循环内再构造一个循环，逐个对所产生的奇数X进行检验，用2到 $\mathbf X - 1$ 之间的所有自然数去除X,如果都不能整除，X就是素数，否则X不是素数。据此可画出图6.4所示的框图。

例6.5有一个古老的问题：100匹马驮100块瓦。大马每匹驮三块，老马每匹驮两块，小马两匹驮一块。问大马、老马、小马各有多少匹？

分析：设大马有 $x$ 匹，老马有 $y$ 匹，小马有 $z$ 匹。依题意可列方程组

$$
x + y + z = 1 0 0
$$

$$
3 x + 2 y + { \frac { z } { 2 } } = 1 0 0
$$

这种未知数个数多于方程个数的方程，叫做不定方程。利用计算机运算速度快的特点，可以用穷举法找出不定方程的解。

在这个问题里，大马的匹数可能是0到33中的一个数，老马的匹数可能是0到50中的一个数，小马的匹数是100与大、老马之和的差。构造一个循环，让大马的匹数从0变化到33。在这个循环中再构造一个循环，让老马的匹数从0变化到50。从这 $3 4 \times 5 1$ 种可能中找出符合 (2)式的三种马的匹数来，就能得到结果。据此可画出图6.5中的框图。

10 FOR $\mathbf { X } { = } 0$ TO 33   
20 FOR $\mathbf { Y } = 0$ TO 50   
30 LET ${ \bf Z } = 1 0 0 - { \bf X } - { \bf Y }$   
40IF $3 * \mathbf { X } + 2 * \mathbf { Y } + \mathbf { Z }$ / $2 = 1 0 0$ THEN PRINT $\mathbf { \ " } \mathbf { X } = \mathbf { \ " }$ ; X, $\mathbf { \ " } \mathbf { Y } = \mathbf { \ " }$ ; Y, $\mathbf { \ " } Z = \mathbf { \ " }$ ; Z   
50 NEXT Y   
60 NEXT X

运行这个程序，屏幕上会显示出

${ \sf X } = 5$ Y=25 Z=70   
$\mathsf { X } = 8$ Y=20 Z=72   
$\mathsf X = 1 1$ Y=15 Z=74   
X=14 Y=10 Z=76   
X=17 Y=5 Z=78   
$\mathsf { X } = 2 0$ Y=0 Z=80   
]

得到7组解。

# 第三节检索与排序

利用计算机的判断功能和它运行时的高速度，可以从大量的数据中很快找到满足人们不同需要的数据，这就是检索。为了满足分析、研究的需要，按照一定的规则为大量数据排出顺序就是所谓的排序。在进行这类处理时，计算机一般要先把数据存放在内存储器里一些事先准备好的存储单元里，用特定的变量来指明数据存放的位置。这类变量就是本节要介绍的下标变量。

# 一、单下标变量与一维数组

用BASIC语言编程序时，是通过变量来进行对内存储器的存取操作的。例如，用LET$\mathbf { A } = 1 0 0$ 把100这个数据存入内存储器中变量A指定的存储单元里，用PRINTX,Y从内存储器中变量X和Y指定的存储单元里取出数据并显示在屏幕上。到目前为止，我们使用过的变量，如A,B,X,Y,F1,F2等,叫做简单变量。简单变量不能反映出数据存储的顺序，即使是像F1,F2这样带有数字的简单变量，对于计算机来说也只是两个不同的变量而已。能反映出数据存储顺序的变量，是下标变量。下标变量由下标变量名和由括号括起来的下标构成，形如A(n)。其中A是下标变量名，下标变量的命名规则与简单变量相同；n是下标变量的下标，下标必须放在圆括号里,可以是数、简单变量或其他算术表达式，也可以是下标变量，如A(1),B(2),X(M),Y(A()),$\mathbf { B } ( \mathbf { X } + 1 )$ 等等。下标表示的是下标变量的序号，因此下标的值只能是0或正整数。如果下标值是小数，计算机会自动取整。

具有相同名称的下标变量的集合叫做数组。数组用来存放一组相互关联的数据。数组用组成它的那些下标变量的名称来命名。例如，由 $\mathrm { H } ( 0 )$ ，H(1),，H(9)等下标变量组成的数组，叫做“数组 $\mathbf { H } ^ { \ast }$ 。属于某一个数组的各下标变量叫做该数组的元素。

存放字符串的数组叫字符串型数组，字符串型数组的元素是字符串型的下标变量。字符串型下标变量名的末尾要有一个 $\$ 8$ ，如 $\mathbf { A } \mathbb { S } ( 0 )$ ， $\mathbf { B } \mathbb { S } ( \mathbf { X } - 1 )$ ， $\mathbf { X } { \mathfrak { S } } ( \mathbf { A } ( 9 ) ) ,$ 等等。

像 A(1), $\mathbf { B } \mathbb { S } ( \mathbf { X } - 1 )$ 这样只有一个下标的下标变量，叫做单下标变量；由单下标变量组成的数组，叫做一维数组。

# DIM $<$ 数组名 ${ } > ( <$ 最大下标值 $>$ ）

计算机遇到DIM语句，就会在内存储器中保留足够的存储单元供存放数组之用。例如，程序中要用数组M存放最多50个数据，就要在程序中实际使用数组M之前用

10 DIM M(49)

来确定数组可以有 M(O)，M(1),，M(49)这 50个数组元素。利用 DIM语句，可以说明多个数组的大小，这时要用逗号把所说明的数组隔开。例如，语句

5 DIM A(100),B\$(80), N(99)

说明程序中要用到数组A,数组B\$和数组N,它们最多分别可以有101，81和100个元素。

使用DIM语句时，每一个数组只能说明一次，否则计算机会给出出错信息

REDIM'D ARRAY ERROR IN xXX指出在第xxx语句发生了重复说明数组大小的错误。

# 本章小结

1．解决实际问题时，如果无法预先知道循环次数，可以用条件语句或条件语句配合GOTO语句构成循环，否则应使用FOR-NEXT语句来构造循环程序。为了使程序条理清楚，应尽量限制GOTO语句的使用。

2．用多重循环处理问题时，要注意循环嵌套的正确性。循环体中使用条件语句时，要正确控制程序转移的方向。

3．数组由下标变量组成。下标表明了该下标变量在数组中的位置。

4．检索数据时，一般先把数据存入一个数组，然后在数组中按规定的条件检查、搜索，从而找到所需的数据。

5．用二重循环为M个数据排序时，外循环变量的值应从1变到 $\mathbf { M } - 1 ,$ ，内循环变量的值每次应从外循环变量当前值加上1变到M。
