# 第八章 音乐与作图

我们使用的微型计算机，不但可以进行数值计算、逻辑判断、文字处理，还可以演奏音乐和绘制图形。

# 第一节让计算机演奏音乐

# 一、发音单元

中华学习机和 AppleⅡ 型计算机里，有一个扬声器，利用它可以发出各种声音来。这两种计算机把内存储器的49200单元作为控制扬声器的开关。通过命令或语句访问这个存储单元时，主机的扬声器就会发出声音来。适当控制发音的频率，就可以让扬声器发出各种乐音。

# 二、产生乐音的机器语言子程序

要让Apple Ⅱ 计算机演奏音乐，必须先执行下面的这个程序。

10 FOR $\mathbf { I } { = } 7 7 0$ TO 790

20 READ A

30 POKEI,A

40 NEXT I

50 DATA 173,48,192,136,208,5,206,1,3,240,9,202,208,245,174,0,3,76,2,3,96计算机执行这段程序后，通过READ语句和POKE语句，把一个产生乐音的机器语言子程序存入了内存储器的 $7 7 0 \sim ~ 7 9 0$ 单元。完成这个准备工作后，就可以运行音乐程序了。

如果使用的是中华学习机，可以省略以上步骤。

（1）从表8.1可以看出，代码的值越小，相应的音（频率）越高。用代码1表示休止符，是因为它表示的音的频率太高，人耳无法听到。音高代码的允许值是 $1 \sim 2 5 5 $ ，超过了这个范围，计算机会出错。

(3）在一个DATA语句中，不宜放置过多的数据，以便查错与修改。

执行 MUSIC语句时，计算机会让扬声器发出由音高代码和音长代码指定的乐音，而且不必事先输入产生乐音的机器语言子程序。

# 第二节用计算机作图

计算机有较强的绘图功能，既可以用字符组成图案，又可以用色块或色点画出更为细致的图形来。

# 一、屏幕显示方式

中华学习机和AppleⅡ机的显示器有三种显示方式，即文本方式、低清晰度图形方式和高清晰度图形方式。开机后，显示器自动进入文本方式。这时，整个屏幕可以显示 24行 $\times 4 0$ 列$\mathtt { \mathtt { = 9 6 0 } }$ 个字符。在文本方式下，可以利用TAB(X)函数控制字符在屏幕上的输出位置，组成图案。但是，这种用字符组成的图案比较粗糙，实际绘图时，常常让显示器进人低清晰度图形方式或高清晰度图形方式。

在低清晰度图形方式下，屏幕上每行可以显示40个色块，整个屏幕最多可以显示 48行。用色块可以组成比较细致的图形。

在高清晰度图形方式下，屏幕上每行可以显示 280个色点，整个屏幕最多可以显示192行。用色点可以在屏幕上描绘出非常细致的图形来。

# 二、高清晰度图形的制作

# 3.绘图

在高清晰度图形方式下，用HPLOT语句可以画出用色点组成的图形来。HPLOT语句有四种格式。

$\textcircled{1}$ HPLOT X , Y   
$\textcircled{2}$ HPLOT X1, Y1 TO X2 , Y2   
$\textcircled{3}$ HPLOT TO X , Y   
$\textcircled{4}$ HPLOT X1 , Y1 TO X2，Y2 TO X3, Y3 TO  TO Xn ，Yn

计算机执行第 $\textcircled{1}$ 种 HPLOT语句时，在屏幕上坐标为 $( \mathbf { X } , \mathbf { Y } )$ 的位置上画出一个点；执行第$\textcircled{2}$ 种HPLOT语句时，在屏幕上从点 (X1,Y1)到点 (X2,Y2)画一条线段；执行第 $\textcircled{3}$ 种HPOLT语句时，从上一次执行HPLOT时画出的最后一点开始到点 $( \mathbf { X } , \mathbf { Y } )$ 画一条线段；计算机执行第$\textcircled{4}$ 种 HPLOT语句时，以（X1,Y1)、（X2,Y2)、…、 $( \mathbf { X } \mathbf { n } , \mathbf { Y } \mathbf { n } )$ 为顶点画出连续的折线。

# 本章小结

这一章介绍了让计算机演奏乐曲与画图的方法。

MUSIC语句可以让中华学习机演奏乐音，把乐曲的每一个乐音转换成音高代码和音长代码后，就可以用MUSIC语句指挥计算机演奏出动听的乐曲来；要让AppleⅡ型计算机演奏乐曲，需要先输入一段机器语言子程序，然后用POKE语句把音高代码存入内存的768单元，把音长代码存入769单元，再用语句CALL770调用产生乐音的子程序；演奏好乐曲的关键是准确输入各个乐音的音高代码和音长代码。用高清晰度图形方式作图时，使用的有HGR语句、HGR2语句、HCOLOR语句和HPLOT语句，画好图形的关键是确定各色点的坐标。

# \*第九章 程序设计方法

本书的第三章到第八章，介绍了用BASIC语言进行程序设计的基本知识。学习了那部分内容后，我们就可以用BASIC语言编写一些程序，让计算机执行它们，从而解决一些实际问题了。BASIC语言只是各式各样计算机语言中最容易学习的一种。用计算机解决实际问题时，还可以使用其他程序设计语言。不同的计算机语言的语法规则可能有很大的差异，但程序设计方法是独立于各式各样计算机语言之上的。掌握了程序设计方法的有关知识，就有了编程序应用计算机解决问题的基础，在此基础之上，熟悉某一种计算机语言后，就可以做到“触类旁通”了。

在介绍程序设计方法之前，我们先分析一下程序的基本结构。

# 第一节程序的基本结构

从前面几章中可以看到，程序是一个用一系列语句描述的用计算机解决问题的过程。由于要解决的实际问题的复杂程度各不相同，编出来的程序也就存在着或大或小、或简或繁的差异。不同的程序设计者为解决同一个问题所编的程序也会存在很大的差异。经过对大量程序的分析可以发现，尽管存在着各种差异，计算机在执行一个程序或一个程序段时，基本上只有三种情况：按语句的排列顺序执行、执行到某一语句时产生分支、重复执行一组语句。按照计算机执行情况的不同来分类，程序或程序段有三种不同的基本结构，即顺序结构、分支结构和重复结构。习惯上把构成整个程序的程序段或者组成一个程序系统的各种程序叫做模块。

# 二、结构化程序

顺序结构、分支结构、重复结构等程序的基本结构都只有一个人口和一个出口。各种基本结构模块可以通过复合与嵌套组成较大的模块。用较小的模块组成较大的模块时，较大的模块仍然只有一个出口和一个入口。如此继续下去，就可以构成整个程序。这种由基本结构模块逐步构造成的程序，就是所谓的结构化程序。可以认为，结构化程序是用基本结构模块通过复合、嵌套而构成的程序。

# 第二节结构化程序设计方法

编程序解决实际问题的过程大致可分三步来进行：分析问题，确定算法；实际编写程序；调试程序。实际编写程序是这一节要讨论的内容。首先看一下结构化程序设计的基本方法。

# 一、自顶向下，逐步求精

我们知道，编写程序是用计算机语言描述对实际问题的处理过程。这与写文章有很多相似的地方。写文章时，一般要先拟出提纲，把整个文章分成几大部分。然后，为每一部分再列出较细的提纲。这样逐步细分下去，直到拟出章、节、段落的细目。最后，按纲目充实以具体内容，经过加工、润色，一篇条理清晰、文字流畅的文章就脱稿了。

结构化程序设计的基本方法是“自顶向下，逐步求精”的方法。这种方法与写文章时采用的方法类似。用自顶向下逐步求精的方法进行程序设计时，先把对一个问题的处理过程粗分为几个模块；然后把每一个模块再细化为几个较小的模块；。这样一步步细化下去，直到每个小模块的处理功能直接用语句来实现，就完成了整个程序设计任务。

在逐步求精的过程中，每一步只是把一个模块细化为几个较小的模块，这种细化过程不容易出错，因而保障了整个程序的正确性。即便出现了错误，由于程序的结构清晰，便于由粗到细逐块查找与修改，而且，某一模块内的错误只需在本模块内进行适当的调整。

为了说明逐步求精的过程，请看一个简单的例子。

# 二、结构化程序设计方法的要点

结构化程序设计的基本方法是“自顶向下，逐步求精”。在具体实践中还要注意以下几个方面的问题。

$\textcircled{1}$ 把整个程序分成若干模块时，每个模块只能有一个人口和一个出口;

$\textcircled{2}$ 要按基本结构进行模块间的连接;

$\textcircled{3}$ GOTO语句只与IF语句配合使用，以构成分支或重复结构模块;

$\textcircled{4}$ 用必要的注释语句区分不同的模块并说明其功能;

$\textcircled{5}$ 把需多次使用的模块设计成子程序的形式;

$\textcircled{6}$ 程序编好后，如确实需要修改，要注意保持程序的模块化结构。

# 三、程序设计的准则

结构化程序设计为我们提供了良好的方法。要设计出结构、性能良好的程序，还需要注意程序设计的一些准则。

# 1.程序的正确性与可靠性

程序的价值首先在于能够运行并正确地解决问题，即在预定的条件下输入必要的数据后，能

得到正确的输出结果。其次，还要求程序必须稳定可靠，在遇到意外情况时能自动进行适当的处理。例如，程序能识别由于误操作输人的不合理的数据，并能采取适当的对策，以确保程序的运行不受干扰。可靠性指的是程序正常工作的能力，一般用平均无故障时间来衡量。

# 2.程序的可读性

所谓可读性，指的是对程序进行阅读与理解的方便程度。程序编好后，一般要经过多次修改才能确保其正确性与可靠性。要修改，就必须先读懂它、理解它。现代计算机的应用程序往往要若干人分工合作来完成。某个人编写的程序，不仅自己能读懂，还要使其他人也能够读懂，这样才便于整个程序系统的调试、组装等工作的顺利进行。为此，应尽可能地在程序中关键的位置上使用注释语句，说明程序模块的功能、变量的作用等问题。

# 3.程序的可维护性

程序运行时可能出现故障。出现故障后检查故障的原因、确定程序中的错误位置、修正程序中的错误等操作需要一定的时间。程序的可维护性的优劣就是以这类操作所需时间的长短来体现的。

# 4.程序的可移植性

程序的可移植性指的是在某一种型号的计算机上编写的程序，转到其他型号的计算机上去运行的能力。

# 5.程序的可扩充性

程序的可扩充性指的是在不改变整体结构的前提下，经过某些处理使程序的功能有所扩充。

# 6.程序的效率

程序的效率一般指程序运行时占有系统资源的多少与程序本身处理功能之间的比率。这里所说的系统资源，包括存储空间、机时等。如果一个不很长的程序在较短的时间内能够解决很复杂的问题，我们就说这个程序的效率比较高。反之，如果一个冗长的程序所能解决的只是比较简单的问题，那么它的效率就是很低的。由于现代计算机的存储容量很大、运算速度很高，为了保证程序的可靠性、可读性、可维护性、可扩充性等指标，在设计程序时可以把进一步提高程序效率的工作放在较次要地位上。

# 第三节程序的调试

编写工作结束以后，程序可能会有不完备的地方，也可能有错误。为确保程序能够顺利地完成预定的任务，还要对程序进行调试。在这里，调试是对程序进行检测、确定错误发生的部位、

排除错误等操作的总称。

# 一、程序中常见的错误

# 1．算法错误

在分析问题、确定算法时出现的失误，会造成算法错误。按照错误的算法编写出来的程序当然无法达到预定的目的。这类错误，计算机运行时检测不出来。要纠正算法错误，必须重新分析问题、确定正确的算法，重新设计程序。

# 2．语法错误

各种程序设计语言都有严格的语法规则。初学者不熟悉这些语法规则或者在输入程序时的失误等，都会造成程序中出现语法错误，使程序不能正常运行。

运行 BASIC程序时，若程序中有语法错误，计算机会在执行到有错误的语句时停下来，并显示出一些信息指明错误的类型与出错语句的行号。

# 3.不违反语法规则的录入错误

如果输入的数据有错误，例如输入小数点时按了逗号键、输入变量名时按错了字母键等。这类错误没有破坏语句的语法结构，程序还会运行下去，但往往会得到错误的结果。计算机不能自动检测出这类错误。

# 4.逻辑错误

由于设计不当，程序中可能会隐含着逻辑错误。某些逻辑错误可以中断程序的运行，例如，进行除法运算时除数为零、试图对负数进行开平方运算等。但多数逻辑错误不违背语法规则，因此，计算机不能自动检测出来。例如，由于计算机计算时产生的误差，使得 $\mathbf { X } { = } 9$ 时，关系式$\mathbf { X } { \hat { \mathbf { \theta } } } 2 { = } 8 1$ 并不成立，导致在程序中用 IF $\mathbf { X } { \hat { \mathbf { \theta } } } 2 = 8 1 $ THEN…这样的语句行处理问题时，永远不会执行THEN后面的语句。查找这样的逻辑错误往往比较困难。

# 二、程序的调试方法

# 1．自底向上逐层调试

我们知道，结构化程序设计的基本方法是自顶向下逐步求精。程序的调试过程应反过来，自底向上逐层调试，即先调试最低层次的模块，确实无误后，再调试由这些最低层次模块构造成的上一个层次的模块。这样逐层调试，直到整个程序完成调试操作。对各模块进行的调试叫做分调。大型软件的分调可以用多台计算机分头进行。分调完成后，要把所有的模块装配起来进行联调。所谓联调就是分调后对整个程序进行的调试。具体调试时，可以采用下面介绍的一些技巧与方法。

# 2.利用BASIC解释程序自动检测语法错误的功能

程序运行中断，屏幕上显示出出错信息时，可以根据出错信息的提示调出有语法错误的语句，分析一下产生错误的原因，修改有关的语句后再继续运行。

# 3.输入适当的初始数据进行程序的试运行

编好程序后，可以先提供一些初始数据让程序进行试运行。所提供的初始数据要有代表性，运行结果应是可预知的，以便利用试运行输出的结果来分析程序的正确性。例如，要调试解一元二次方程 $a x ^ { 2 } + b x + c = 0$ 的程序，可以把2、9、4作为方程的系数输入，看看输出的两个根是否为 $- 0 . 5$ 和-4;把2、4、2作为方程的系数输入，看看输出的两个根是否都是-1;把7、2、5作为方程的系数输入，看看输出的结果是否表示无实根。若发现错误，可以到相应的模块中去查找。需要说明的是，利用这种方法往往可以找出程序中的错误，但不能保证程序没有错误，所以还要配合其他方法来调试。

# 4.按 $\mathbf { C } \mathbf { m } + \mathbf { C }$ 键中断程序的运行

发现程序可能有逻辑错误时，如运行时间超长、不能正常结束等，可以按 $\mathrm { C t r l } { + } \mathrm { C }$ 键使程序停止运行。然后用立即执行方式的命令，如PRINT命令，检查有关变量的值。通过对有关变量当前值的分析，可以找出错误的根源。例如，程序运行到一个循环内部不能结束时，可以用上述方法检查循环控制变量的当前值，了解一下循环体执行了多少次，然后进一步确定出现错误的原因。

中断程序的运行以后，经过检查可以用立即执行方式的赋值语句改变某些变量的值，然后用CONT命令使程序再继续运行，进一步观察程序的运行情况。如果修改了程序中的语句，必须用RUN命令使程序的运行从头开始。

# 5.设置断点

在程序中插入一些STOP语句，可以使计算机遇到STOP语句时中断程序的运行。插入这些STOP 语句，相当于在程序中设置了一些断点。用设置断点的方法，可以对程序分段调试。如未发现错误，用CONT命令可以使程序继续运行。

断点的位置应设在程序中各个模块的末尾，以便有计划地逐块检查程序的运行状态。调试完毕，应删除那些STOP语句，把断点取消。

# 6.跟踪程序的运行

# 本章小结

这一章主要介绍了程序的基本结构、结构化程序设计方法和程序的调试等方面的内容。在介绍用 BASIC语言进行程序设计的章节里，已经渗透了结构化程序设计的思想，这里做了进一步的概括与总结。程序的模块化是结构化程序的一个重要特征，需加深对这一点的理解。结构化程序设计方法是本章的重点，要注意在今后的程序设计实践中应用，形成良好的程序设计风格。

1．任何复杂、庞大的程序，都可以分割成具有顺序结构、分支结构或者重复结构的若干基本结构模块。基本结构模块只有单一的入口和单一的出口。用基本模块经过复合、嵌套组成的程序，叫做结构化程序。

2.结构化程序设计的基本方法是自顶向下逐步求精。进行结构化程序设计时，要注意程序的模块化和结构的基本特征。在程序设计中，除了程序的正确性和可靠性以外，可读性、可维护性和可移植性是首先要考虑的标准，同时也要考虑程序的效率。

3．程序运行时会出现各式各样的错误，实际运行时要特别注意屏幕上出现的提示信息，看懂后再着手进行修改等操作。调试的方法与技巧要经过反复实践才能掌握，因此要注意在实际操作中积累经验。
