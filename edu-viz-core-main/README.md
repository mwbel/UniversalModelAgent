# AhaTutor v2.0

> **基于 A2UI 协议的项目驱动智能学习系统**

---

## 项目愿景

AhaTutor 不是一个传统的"AI 家教"，而是一个**项目驱动的学习操作系统**。

我们相信，最好的学习不是"上课"，而是**带着想完成的目标来，在创造过程中掌握知识**。

用户不会说"教我物理"，而是说：
- "我想做一个弹簧振子模拟器"
- "我想写一个 2D 游戏物理引擎"
- "我想理解 Transformer 并复现一个小模型"

AhaTutor 会：
1. **拆解目标** → 生成学习路径树
2. **评估掌握** → 识别知识缺口
3. **可视化教学** → 用交互式组件解释关键概念
4. **推进项目** → 把学习转化为实际产物
5. **回溯整合** → 让知识服务于目标

最终，用户不只是"学会了"，而是**真正完成了想做的东西**。

## 核心技术特色

### 🎨 A2UI 协议驱动的交互式教学

基于 Google 开源的 **A2UI (Agent-to-UI) 协议**，实现 AI 直接生成交互式可视化组件的能力。

- LLM 在流式回答中嵌入 A2UI JSONL 消息
- 前端实时渲染学科可视化组件（物理模拟器、几何工具、数学图形等）
- 用户与组件的交互回传给 AI，形成教学闭环

**这意味着**：AI 不只能"说"，还能"画"和"演示"，并根据你的操作调整教学策略。

### 🧩 插件化学科知识体系

每个学科领域都是一个独立插件，包含：
- **知识库**：向量数据库存储学科知识
- **组件库**：React 可视化组件集合
- **能力清单**：该学科可调用的交互式教学工具

插件可以独立开发、更新、组合，形成跨学科的学习体验。

### 🤖 多智能体协作架构（规划中）

未来将采用专家智能体分工协作：

| 智能体 | 职责 |
|--------|------|
| **LearningOrchestrator** | 总控，维护学习状态，决定下一步 |
| **CurriculumArchitect** | 根据目标生成学习路径树 |
| **MasteryJudge** | 评估掌握度，识别知识缺口 |
| **Tutor** | 主教学智能体，负责解释和引导 |
| **Visualizer** | 决定何时调用哪个 A2UI 组件 |
| **IntegrationAgent** | 把子知识回溯整合到项目目标 |
| **ProjectBridge** | 把学习成果转化为项目实现步骤 |

## 技术架构

### 前端技术栈
- **React 19** + **TypeScript** + **Vite**
- **@a2ui/react** - A2UI 渲染引擎
- **Zustand** - 状态管理
- **Tailwind CSS** - 样式系统
- **Three.js** - 3D 可视化支持

### 后端技术栈
- **FastAPI** - 异步 Web 框架
- **OpenAI API** - LLM 驱动
- **SQLite** + **向量数据库** - 知识存储
- **SSE** - 流式响应协议

### 设计理念
**"The Illuminated Mind"** - 拒绝传统 EdTech 的仪表盘美学，采用高端编辑排版风格。

- 大量留白、非对称布局、玻璃态分层
- 营造"阳光图书馆"的沉浸感
- 让 UI 消失，只留下知识的清晰呈现

详见 [设计规范](temp/原型/DESIGN.md)

## 快速开始

### 前端开发
```bash
cd frontend
npm install
npm run dev
```

访问 `http://localhost:5174` 查看主应用  
访问 `http://localhost:5174/?gallery=1` 查看组件预览

### 后端开发
```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload
```

API 文档：`http://localhost:8000/docs`

## 主要创新点和难点

### 创新点

1. **A2UI 协议在教育场景的应用**
   - 将 Google 开源的 Agent-to-UI 协议引入智能教学领域
   - 实现 LLM 直接生成交互式可视化组件，突破传统"文字+图片"的教学局限

2. **项目驱动的学习范式转变**
   - 从"课程目录"转向"目标树"，以用户想完成的项目为起点
   - 动态生成个性化学习路径，知识学习与项目推进双向闭环

3. **多智能体协作教学架构**
   - 专家智能体分工协作：目标拆解、掌握度评估、可视化教学、知识整合
   - 编排器统一调度，避免单一 AI 的能力边界问题

4. **插件化学科知识体系**
   - 学科知识库与可视化组件解耦，支持独立开发和组合
   - 跨学科项目可自动调用多个插件协同教学

### 技术难点

1. **A2UI 消息的智能生成与流式协同**
   - LLM 需在流式输出中准确嵌入符合 A2UI v0.8 规范的 JSONL 消息
   - 文本解释与组件渲染的时序同步，保证教学连贯性

2. **学习路径的动态生成与个性化**
   - 根据用户目标自动拆解知识依赖树，粒度需适配用户水平
   - 实时调整路径：根据掌握度评估结果动态增删节点

3. **多智能体的状态管理与协调**
   - 全局学习状态（目标树、掌握度、项目进度）的一致性维护
   - 智能体间的任务切换与上下文传递，避免信息丢失

4. **知识掌握度的准确评估**
   - 区分"记住"与"理解"，识别用户的误区和盲点
   - 开放式回答的自动判定，避免过度依赖选择题

5. **学习成果到项目产物的转化**
   - 把抽象知识映射为具体实现步骤
   - 保证用户"学完能做"，而非"学完就忘"

## 预期成果形式

### 1. 系统产品
- **Web 应用**：可部署的完整学习系统，支持多用户、多会话
- **插件生态**：至少覆盖 3-5 个学科领域（物理、数学、编程、数据科学等）
- **组件库**：20+ 交互式可视化组件（模拟器、图形工具、代码编辑器等）

### 2. 用户学习成果
- **项目作品集**：用户通过系统完成的真实项目（网页、模拟器、小工具等）
- **能力地图**：可视化的知识掌握图谱，展示学习路径和掌握度
- **学习报告**：每个项目的学习过程记录、知识点覆盖、时间投入分析

### 3. 技术文档与论文
- **系统架构文档**：多智能体协作框架、A2UI 集成方案、插件开发规范
- **研究论文**：项目驱动学习的效果评估、A2UI 在教育场景的应用研究
- **开源贡献**：向 A2UI 社区反馈教育场景的协议扩展建议

### 4. 数据与评估
- **用户学习数据**：学习路径、掌握度变化、组件交互行为
- **效果对比实验**：与传统 AI Tutor 的学习效率、知识留存率对比
- **插件效果分析**：不同学科可视化组件对理解提升的量化评估
