# AhaTutor v2.0 UI/UX 演进方案：从对话到项目驱动学习

## 1. 当前页面理解
*   **整体布局：** 采用了三栏式的非对称布局。
    *   **左侧 (Library/History):** 沉浸式的会话历史管理，使用了极轻的背景色区分。
    *   **中间 (Main Stage):** 核心对话区域，具有大量的留白和呼吸感，背景带有流动的光影纹理。
    *   **右侧 (Insight/Gallery):** 洞察与组件展示区，采用玻璃态（Glassmorphism）浮层，用于呈现 A2UI 组件和总结建议。
*   **视觉延续点：** 
    *   Newsreader 衬线体用于内容，Manrope 无衬线体用于 UI 控件。
    *   `#faf9f5` 的温暖底色与 `#182544` 的深靛蓝文字。
    *   非线性的边框处理（优先使用色阶和阴影）。
    *   “金色”作为高光时刻（顿悟、完成）的唯一强调色。

---

## 2. 总体 UI 演进策略
*   **演进逻辑：** 保持“聊天”为核心交互介质，将“学习树”作为聊天的**上下文导航**，将“项目进度”作为聊天的**最终反馈**。
*   **去工具化：** 学习树不应表现为死板的树状图，而应像是一张**知识地图**或**思维涟漪**，强调连接感而非层级感。
*   **视觉亮点：** 节点“掌握”时的金色晕染效果，以及 Agent 工作流像微光闪烁一样的状态提示。

---

## 3. 功能放置与布局调整建议
保持现有结构，通过“插槽扩展”实现功能注入：

| 功能模块 | 建议位置 | 设计原因 |
| :--- | :--- | :--- |
| **项目目标输入** | 中间区域（对话流起始） | 它是对话的开启点，应以大面积留白和精美排版呈现。 |
| **学习目标树** | 左侧栏（顶部切换或原历史列表上方） | 作为全局导航，引导用户了解“我在哪，要去哪”。 |
| **当前节点状态/进度** | 右侧栏顶部 | 实时反馈当前学习的微观状态。 |
| **Agent 工作流状态** | 聊天输入框上方 / 右侧栏底部 | 轻轻提示系统正在进行的思考，不干扰主内容。 |
| **A2UI 组件展示** | 右侧栏主区域 / 对话流中嵌入 | 保持现有逻辑，右侧作为深层交互区。 |
| **掌握度判断/测评** | 对话流中的特殊卡片 | 作为教学逻辑的一部分，随对话自然出现。 |

---

## 4. 新增 UI 模块描述
1.  **知识涟漪面板 (Knowledge Tree):** 位于左侧。节点使用极细线条连接，当前节点带有微弱的呼吸光效。
2.  **顿悟刻度条 (Project Progress):** 位于右侧栏顶部。使用极细的金线表示，而非粗壮的进度条。
3.  **Agent 思考波束 (Workflow Status):** 在输入框上方，显示一串微小的、有节奏闪烁的文字（如：*“正在生成测评题...”*），保持安静。
4.  **掌握度确认卡片:** 对话流中的 UI。提供三个选项：*“我已了解”、“挑战一下 (测评)”、“开始学习”*。

---

## 5. 用户交互流程 (关键节点反馈)
*   **进入与生成：** 用户输入目标后，中间区呈现“目标解析中”的排版动画，左侧树逐级展开。
*   **切换节点：** 用户点击左侧树节点，中间对话流平滑滚动或切换上下文，并伴随标题提醒：“我们现在开始探讨 [节点名称]”。
*   **掌握反馈：** 测评通过后，右侧进度条金线延长，左侧对应节点变为金色实心点。
*   **SSE 流式教学：** 文字保持 Newsreader 字体优雅流出。若涉及组件，右侧栏自动开启或更新。

---

## 6. 状态 UI 设计
*   **生成中:** 采用骨架屏，但带有模糊的金色光晕效果。
*   **空状态:** 中间区域仅显示一行富有诗意的引导语，如 *“你想通过学习创造什么？”*
*   **错误状态:** 不要使用红色的警告框。使用深靛蓝的提示文字，并提供一个带有金色下划线的“重试”链接。

---

## 7. 第一阶段 UI MVP (10 天目标)
*   **必须做：**
    1.  左侧学习树的静态展示与节点选中态。
    2.  右侧项目进度金线的逻辑联动。
    3.  对话流中嵌入“掌握度判断”选择卡片。
*   **可后置：**
    1.  复杂的节点回溯总结动画。
    2.  多 Agent 详细状态流转图。
*   **Demo 亮点：** 展示从“输入目标”到“生成树”再到“节点变金（掌握）”的完整链路。

---
**注意事项：** 永远不要让页面看起来像 Jira 或飞书。保持它作为“发光的思想”这一沉浸空间的纯粹性。
