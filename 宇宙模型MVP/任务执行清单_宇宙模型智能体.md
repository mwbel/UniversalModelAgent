# 宇宙模型智能体任务执行清单

## 1. 文档信息

- 项目名称：宇宙模型智能体
- 文档版本：v1.1
- 文档日期：2026-05-24
- 关联文档：`PRD_宇宙模型智能体.md`
- 文档用途：用于 MVP 阶段任务拆解、排期、执行跟踪
- 当前实施说明：MinerU OCR 服务与多版本 RAG 服务已部署到外部服务器，执行重点从“自建底层能力”调整为“服务接入、能力评测、统一编排与结果验收”

## 2. 执行目标

本清单用于将 PRD 拆解为可执行任务，围绕以下三个核心能力推进：

1. 智能 AI 问答
2. 中英文天文学与数学 RAG 知识库
3. 基于代码的交互式天文学可视化

目标是在 MVP 周期内完成一个可演示、可测试、可迭代的产品雏形。

## 3. 执行原则

- 优先闭环：先打通“提问 -> 检索 -> 回答 -> 可视化”的主链路
- 优先 MVP：先做最小可用版本，再做体验增强
- 优先高价值场景：优先覆盖最能体现产品差异化的场景
- 优先可验证：每个阶段必须有明确产出物和验收标准

## 4. 阶段划分总览

### 阶段 0：项目启动与方案确认

目标：明确产品范围、技术路线、交付边界。

### 阶段 1：基础架构与开发环境搭建

目标：完成项目骨架、基础工程和运行链路。

### 阶段 2：RAG 知识库首版建设

目标：建立首版可检索、可引用的中英文知识库。

### 阶段 3：AI 问答能力开发

目标：实现基于大模型与 RAG 的对话问答能力。

### 阶段 4：交互式可视化模块开发

目标：完成首批核心天文学现象或概念的交互式演示。

### 阶段 5：问答与可视化联调

目标：实现问答结果与可视化的自然联动。

### 阶段 6：测试、优化与演示发布

目标：完成验收、演示准备和后续迭代输入。

## 阶段 2：RAG 知识库首版建设

### 2.0 建立 OCR 文档转 Markdown 预处理流程

- 任务内容：
  - 盘点现有 OCR 书籍与资料格式
  - 确定批量转换目标格式为 Markdown
  - 接入已部署的 MinerU 服务用于批量转换
  - 清理常见 OCR 噪声，如断行、乱码、分页残留、公式错位
  - 建立转换结果抽检机制
- 输出物：
  - OCR 转 Markdown 处理流程
  - 首批转换后的 Markdown 文档样本
- 验收标准：
  - 主要资料可稳定转为可读 Markdown
  - 转换结果可支撑后续知识清洗与入库

### 2.1 收集首批知识资料

- 任务内容：
  - 收集中文天文学资料
  - 收集英文天文学资料
  - 收集数学基础资料
  - 收集中英术语对应资料
  - 筛选授权清晰、可信度高的来源
- 输出物：
  - 首批知识源清单
- 验收标准：
  - 每个核心主题至少有 2 到 3 份有效来源

### 2.2 建立知识入库规范

- 任务内容：
  - 以前置 Markdown 文档形态为基础制定规范
  - 定义文档清洗规则
  - 定义文本切分策略
  - 定义元数据字段
  - 定义标签体系
  - 定义来源可信度等级
- 输出物：
  - 知识入库规范文档
- 验收标准：
  - 新资料可以按统一标准入库

### 2.3 开发文档解析与入库脚本

- 任务内容：
  - 优先支持 Markdown 导入
  - 按需要兼容 TXT、PDF 等基础格式导入
  - 实现文本提取与清洗
  - 实现切片与元数据写入
  - 输出结构化入库数据
- 输出物：
  - 文档处理脚本
- 验收标准：
  - 至少能稳定处理首批资料

### 2.4 建立向量索引与检索流程

- 任务内容：
  - 接入已部署的多版本 RAG 服务
  - 统一不同 RAG 服务的输入输出结构
  - 建立多版本效果对比流程
  - 选择首期默认 RAG 路由策略
  - 保留结果重排序或后处理能力
- 输出物：
  - 首版 RAG 接入与评测服务
- 验收标准：
  - 对核心问题能稳定返回相关知识片段与引用结果

### 2.5 构建测试问题集

- 任务内容：
  - 为每个主题整理测试问题
  - 包括中文问题、英文问题和中英混合问题
  - 包括定义题、解释题、推导题、现象题
- 输出物：
  - 检索与问答测试题库
- 验收标准：
  - 可用于回归测试与质量评估

## 阶段 3：AI 问答能力开发

### 3.1 接入大模型服务

- 任务内容：
  - 配置模型调用
  - 配置系统提示词
  - 配置多轮对话上下文策略
  - 配置中英文输出能力
- 输出物：
  - 可调用的大模型问答服务
- 验收标准：
  - 模型可稳定返回基础回答

### 3.2 实现 RAG 问答链路

- 任务内容：
  - 用户问题预处理
  - 调用 RAG 服务召回知识片段
  - 构造带上下文提示词
  - 生成带引用回答
  - 返回引用片段与来源信息
- 输出物：
  - RAG 问答 API
- 验收标准：
  - 回答质量优于无检索直答

### 3.3 设计回答模板

- 任务内容：
  - 设计概念解释模板
  - 设计现象解释模板
  - 设计公式推导模板
  - 设计英文资料解释模板
- 输出物：
  - 回答模板与提示词策略
- 验收标准：
  - 回答结构统一、可读性强

### 3.4 增加不确定性处理机制

- 任务内容：
  - 当检索结果不足时进行提示
  - 当问题超出知识范围时进行提示
  - 当问题存在争议时进行边界说明
- 输出物：
  - 安全回答策略
- 验收标准：
  - 减少编造式回答

### 3.5 实现对话页基础功能

- 任务内容：
  - 输入框
  - 消息列表
  - 回答流式展示
  - 来源引用展示
  - 推荐可视化入口展示
- 输出物：
  - 对话页 MVP
- 验收标准：
  - 用户可完成完整提问与查看回答流程

## 阶段 4：交互式可视化模块开发

### 4.1 确定首批可视化模块名单

- 任务内容：
  - 从 PRD 场景中选 3 到 5 个模块
  - 定义每个模块的教学目标
  - 定义每个模块的输入参数与交互方式
- 建议首批模块：
  - 日食/月食演示
  - 行星逆行演示
  - 开普勒第三定律关系图
  - 哈勃定律图表
- 输出物：
  - 可视化模块需求说明
- 验收标准：
  - 每个模块都有明确教育价值和交互目标

### 4.2 搭建可视化组件框架

- 任务内容：
  - 抽象统一容器组件
  - 统一参数面板样式
  - 统一说明区与图例区
  - 统一播放、暂停、重置交互
- 输出物：
  - 可视化基础组件框架
- 验收标准：
  - 后续模块可以复用组件能力

### 4.3 开发首批可视化模块

- 任务内容：
  - 实现轨道关系演示
  - 实现遮挡现象演示
  - 实现图表类关系演示
  - 加入参数控制与说明文案
- 输出物：
  - 3 到 5 个可运行可视化模块
- 验收标准：
  - 用户能通过交互观察现象变化过程

### 4.4 完成科学说明校验

- 任务内容：
  - 校对可视化中的术语与说明
  - 校验参数变化与现象逻辑是否一致
  - 校验与问答回答的一致性
- 输出物：
  - 可视化校验记录
- 验收标准：
  - 不出现明显科学性错误

## 阶段 5：问答与可视化联调

### 5.1 建立问题分类与路由规则

- 任务内容：
  - 判断哪些问题需要推荐可视化
  - 判断哪些问题需要优先术语解释
  - 判断哪些问题需要优先推导式回答
- 输出物：
  - 问题路由规则
- 验收标准：
  - 不同类型问题能进入合理处理链路

### 5.2 实现问答结果到可视化跳转

- 任务内容：
  - 在回答中返回关联模块标识
  - 前端显示“查看演示”入口
  - 支持跳转到对应可视化页面
- 输出物：
  - 问答联动机制
- 验收标准：
  - 用户能从回答一键进入相关演示

### 5.3 实现上下文参数透传

- 任务内容：
  - 根据问题内容给可视化模块设置默认参数
  - 支持从问题语义中提取场景信息
  - 支持可视化与对话文案一致
- 输出物：
  - 参数联动能力
- 验收标准：
  - 用户进入可视化后不需要从零配置
