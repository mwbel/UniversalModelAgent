# AhaTutor v2.0 UI 设计要素清单

基于 "The Illuminated Mind" 风格与 Project-Based Learning (PBL) 核心需求，以下是各页面及区域需要设计的 UI 要素。

## 1. 全局布局框架 (Layout Framework)
*   **左侧导航栏 (Knowledge & Navigation):**
    *   [ ] 学习目标树 (Knowledge Tree) 容器：非线性连接的节点布局。
    *   [ ] 历史会话 (History) 切换入口：与学习树的平滑切换逻辑。
    *   [ ] 插件/工具管理入口：底部轻量化图标。
*   **中间主舞台 (Main Stage - Conversation):**
    *   [ ] 沉浸式对话流：Newsreader 字体排版，超大留白。
    *   [ ] 动态输入框：支持文件拖入、目标输入的扩展态。
    *   [ ] 背景流体纹理：随学习状态（生成中、已掌握）轻微变化的动态光影。
*   **右侧洞察栏 (Insight & Progress):**
    *   [ ] 玻璃态 (Glassmorphism) 浮层容器。
    *   [ ] 顶部项目进度条 (Project Goldline)：极细金色丝线设计。
    *   [ ] A2UI 组件渲染容器：支持高度自适应与滚动。

---

## 2. 关键功能模块 (Functional Modules)

### A. 项目目标输入 (Goal Intake) - 中间区
*   [ ] **引导态 UI:** 一行富有诗意的引导语（如：“你想通过学习创造什么？”）。
*   [ ] **解析态排版:** 输入目标后，系统自动拆解时的排版动画（大号标题 + 渐显的副标题）。
*   [ ] **生成中状态:** 金色粒子流动的骨架屏。

### B. 学习目标树 (Knowledge Tree) - 左侧区
*   [ ] **节点样式:** 
    *   未解锁态：浅灰色细圆环。
    *   学习中/当前态：深靛蓝实心圆 + 呼吸晕染效果。
    *   已掌握态：金色实心点。
*   [ ] **连接线:** 极细的、具有有机感（非直角）的曲线连接。
*   [ ] **交互态:** 点击节点时的上下文切换提醒 UI。

### C. 教学与掌握度 (Learning & Mastery) - 对话流
*   [ ] **掌握度判断卡片:** 嵌入对话的三个选项（了解/测评/学习），卡片带有轻微阴影和金色下划线。
*   [ ] **测评卡片 (Assessment):** 结构化的题目展示，选择/填空交互，反馈态（对/错）需符合安静的基调。
*   [ ] **A2UI 触发态:** 对话中出现“查看可视化详情”的引导按钮。

### D. Agent 状态 (Workflow Status) - 输入框上方/右侧底部
*   [ ] **思考波束 (Thinking Beam):** 极细的闪烁文字流，显示当前 Agent 正在进行的任务（如：*Analyzing goal...*）。
*   [ ] **多 Agent 协作标识:** 极简的图标图标阵列，轻微闪烁表示活跃。

---

## 3. 状态与反馈 UI (States & Feedback)
*   [ ] **顿悟时刻 (Aha! Moment):** 全屏或局部轻微的金色光晕扩散。
*   [ ] **节点完成总结 (Integration):** 覆盖在对话流上的轻量总结层，展示“已解锁新能力”。
*   [ ] **空状态 (Empty States):** 保持艺术性的插画或文字留白。
*   [ ] **错误状态:** 移除红色警告，改用深靛蓝文字提示 + 金色重试按钮。

---

## 4. 移动端适配要素 (Mobile Adaptation)
*   [ ] **底部抽屉 (Bottom Sheet):** 用于承载学习树和项目进度。
*   [ ] **吸底输入框:** 保持简洁，不遮挡最后一句对话。
*   [ ] **顶部折叠状态栏:** 仅显示当前节点名称与项目进度。

---
**设计原则提醒：** 每一处新增 UI 都应像是在一张精美的纸张上落笔，保持呼吸感，避免工具堆砌。
