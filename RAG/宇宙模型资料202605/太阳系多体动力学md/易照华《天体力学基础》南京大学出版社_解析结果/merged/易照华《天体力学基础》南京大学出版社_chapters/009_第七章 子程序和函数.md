# 第七章 子程序和函数

# 第一节子程序及子程序的调用

用BASIC语言设计较复杂的程序时，还可以采用子程序技术，即把一些功能相对独立的，或者解决问题时需多次使用的程序段写成子程序的形式，需要时用特殊的语句调用有关的子程序。请看下面的例题。

例7.1编一个程序，输出自然数 $\mathbf { N } = 1$ ，2,…，M的各数以及它们的平方、立方、各数的和、各数平方的和、各数立方的和。要求按下面的表格输出（以 $\mathbf { M } = 5$ 为例):

\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*   
\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*   

<table><tr><td>N</td><td>N^2 N^3</td></tr></table>

\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*

1 1 1   
2 4 8   
3 9 27   
4 16 64   
5 25 125

15 55 225分析：为了输出表中各行数据，可编程序如下。

程序一：  
10 LET $\mathbf { S } 1 = 0 \ : \ \mathrm { L E T } \ \mathbf { S } 2 = 0 \ : \ \mathrm { L E T } \ \mathbf { S } 3 = 0$   
20 INPUT $" \mathrm { \bf ~ M } = "$ ; M  
30 PRINT $" \mathrm { \bf N ^ { \prime \prime } }$ ， $" \mathbf { N } \ \cdot \ 2 "$ ，" N^3"  
40 FOR $\mathbf { N } = 1$ TO M  
50 LET $\mathbf { S } 1 = \mathbf { S } 1 + \mathbf { N }$   
60 LET $\mathbf { S } 2 = \mathbf { S } 2 + \mathbf { N } \hat { \mathbf { \Omega } } 2$   
70 LET $\mathbf { S } 3 = \mathbf { S } 3 + \mathbf { N } \hat { \mathbf { \Omega } } 3$   
80 PRINT N, N^2, N^3  
90 NEXT N  
100 PRINT S1, S2, S3  
110 END

为了输出表格中的三条用40个\*号组成的分隔线，可以在程序中插入三个PRINT语句

25 PRINT \*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\* \*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*

35 PRINT "\*\*\*\*\*\*\*\*\*\*\*\*\* \*\*\*\*\*\*\*\*\*\*\*\*\*\*

95 PRIN\* \*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*\*为避免键入120个 $^ *$ 号的麻烦，也可以用

FOR $\mathbf { I } = 1$ TO 40 PRINT "\* ";   
NEXT I   
PRINT

这样的循环程序来完成输出分隔线的任务。但是，这需要在程序中重复安排三个相同的程序段。如果表格需要更多的分隔线，就要多次重复输入这种相同的程序段。下面我们来看一看如何用子程序的方法解决这个问题。

程序二:

20 INPUT $\mathbf { \ " } \mathbf { M } = \mathbf { \ " }$ ; M   
25 GOSUB 200

90 NEXT N   
95 GOSUB 200   
100 PRINT S1, S2, S3   
110 END

210 FOR $\mathbf { I } = 1$ TO 40   
220 PRINT "\* ";   
230 NEXT I   
240 PRINT   
250 RETURN   
在程序二里，语句行 $1 0 \sim ~ 1 1 0$

这段程序，叫做主程序，语句 $2 0 0 \sim ~ 2 5 0$ 这段程序叫子程序。25、35和95行的语句叫做GOSUB语句，也叫转子语句。语句250叫做RETURN语句，也叫返回语句。GOSUB语句的格式是

GOSUB $<$ 子程序起始行号 $>$

RETURN语句的格式是

RETURN

计算机执行GOSUB语句时，先记录下这个GOSUB语句的下一个语句的位置，然后转到子程序的起始语句行去执行，这个过程叫做调用子程序。执行子程序遇到RETURN语句时，计算机会返回本次执行GOSUB语句时所记录的下一个语句去继续运行。

在这个例子里，程序二的运行情况如下：

$\textcircled{3}$ 执行语句25,记下下面那条可执行语句的行号30后转到语句200;  
$\textcircled{4}$ 调用语句行200开始的子程序，输出分隔线，遇到语句250后返回语句30继续执行;  
$\textcircled{5}$ 执行语句30,输出表头的文字;  
$\textcircled{6}$ 执行语句35,记下下面那条可执行语句的行号40后转到语句 200;  
$\textcircled{7}$ 调用语句行200开始的子程序，输出分隔线，遇到语句250后返回语句40继续执行；  
$\textcircled{8}$ 执行 $4 0 \sim ~ 9 0$ 行的那段循环程序，计算各数的和、平方和、立方和并输出各数、各数的平方、各数的立方;

一般说来，子程序应写在主程序的END语句之后，最好用一个REM语句当作这个子程序的起始语句，用来指明这个子程序的功能。子程序中至少要有一个RETURN语句，否则计算机无法返回主程序。

在主程序中，调用子程序之前与从子程序返回以后，应安排适当的变量完成主程序与子程序之间数据的传递任务。

例7.2有24支足球队参加锦标赛，如果采用单循环制，即每队要与其他23个队各比赛1场，组委会应安排多少场比赛？

分析：要解决这个问题，可以借用高中数学第三册里的公式 m!×(n-m）！，用24取代n,用2取代 $m$ ,求出 24个队中每次取两个队的所有组合数。

程序：

10 LET $\mathbf { X } = 2 4$   
20 GOSUB 500  
30 LET $\mathbf { A } = \mathbf { Y }$   
40 LET $\mathbf { X } { = } 2 4 - 2$   
50 GOSUB 500  
60 LET $\mathbf { N } { = } \mathbf { A } / ( 2 { * } \mathbf { Y } )$   
70 PRINT"应安排"； $\mathbf { N }$ ;"场比赛"  
80 END  
500 REM 求X!的子程序  
510 LET $\mathbf Y = 1$   
520 FOR ${ \bf I } = 2$ TO X  
530 LET $\mathbf { Y } = \mathbf { Y } * \mathbf { I }$   
540 NEXT I  
550RETURN

在编写较复杂的程序时，还可以用GOSUB语句代表某个程序模块，先设计好主程序，然后再分别考虑各子程序的设计。这样可以使程序的结构与功能一目了然，便于设计又便于调试。

例7.3有36名考生，参加数、理、化、语文、外语等5科考试。设计一个程序，统计各考生的总分，根据总分从高到低的顺序排出名次，并按名次输出各考生的姓名、各科成绩和总分。

分析：每个考生的数据有姓名和各科成绩等6项内容，可以用一个数组N\$存放姓名，用一个二维数组S存放各科成绩。由于问题比较复杂，先设计主程序。

主程序：

100 DIM N\$(36), S(36 ,5)

110 GOSUB $1 0 0 0 : \mathbf { R E M }$ 输人数据  
120 GOSUB $2 0 0 0 : \mathbf { R E M }$ 计算总分  
130 GOSUB 3000:REM 排出名次  
140 GOSUB 4000:REM 输出结果  
150 END  
160 DATA 李军,100,98,87,90,79,王莉，89,72,90,95,97,孙小伟，98,97,92,80,93  
170 DATA 陈媛,99,89,95,98,93,丁一川，80,89，70,66,58,简志杰，98,78,62,80,71  
180 DATA ·… (以下从略)  
然后分别设计各子程序。先设计输人姓名和各科成绩的子程序。  
1000 REM 输人数据的子程序  
1010 FOR $\mathbf { I } = 1$ TO 36  
1020 READ N\$(I)  
1030 FOR $\mathbf { J } = 1$ TO 5  
1040 READ S(I , J)  
1050 NEXT J  
1060 NEXT I  
1070 RETURN

执行这个子程序时，把每次从DATA语句中读人的姓名存人数组NS,把姓名后面的5个成绩存入二维数组S。返回主程序时，数组 $\mathbf { N \mathbb { S } }$ 中就输入了36个考生的姓名，数组S中就输入了一个36行5列的数表 (序号是0的那一列暂时空着)。

其次，设计计算每个考生总分的子程序。用数组S中第二个下标是0的那些元素存放每个考生的总分(在此之前，未给它们赋过值，所以 S(1,0)、S(2,0)、、S(36,0)的初值为0)。

2000 REM 计算总分的子程序   
2010 FOR $\mathbf { I } = 1$ TO 36   
2020 FOR $\mathbf { J } = 1$ TO 5   
2030 LET ${ \bf S } ( \mathrm { { \bf { I } } } , 0 ) { = } { \bf S } ( \mathrm { { \bf { I } } } , 0 ) { + } { \bf S } ( \mathrm { { \bf { I } } } , \mathrm { { \bf { J } } } )$

2050 NEXT I   
2060RETURN

执行这个子程序时，每次把数表中第I行第1至5列的分数加起来存人第0列。返回主程厅时，36名考生各自的总分就存入了S数组里的数表每行的第0列。

然后设计排名次的子程序。排序时比较每个考生的总分，如果需要交换位置，同时还要交换各科成绩和姓名的存放位置。

3000 REM 排名次的子程序

3060 NEXT K   
3070 LET $\mathbf { T } \mathbb { S } \mathbf { = } \mathbf { N } \mathbb { S } ( \mathrm { I } )$ : LET $\mathbf { N S ( } \mathbf { I } ) { = } \mathbf { N S ( } \mathbf { J } )$ : LET $\mathbf { N S ( J ) } = \mathbf { T } \mathbb { S }$

3090 NEXT I   
3100 RETURN

执行语句3030时，比较存放在数表第0列里各行数据的大小，如果第I行比第J行大，就转到语句3080继续比较下一行，否则，先由语句3040到3060构成的循环把第I行和第J行的所有数据互换存放位置，再由语句3070把相应的考生姓名调换过来，然后再继续比较下一行。返回主程序时，数组S里存放的数表的第0列就排好了总分的顺序，每行中各列的各科成绩和数组NS里的考生姓名也作了相应的调整。

# 第二节BASIC函数

# 一、标准函数

在数值计算中，经常需要使用数学函数，如对数函数、三角函数以及本书前面用过的求算术平方根函数 SQR(X)、取整函数 $\mathbf { I N T } ( \mathbf { X } )$ 、随机数函数RND(X)、绝对值函数ABS(X)等。在BASIC语言里，这些函数叫做标准函数或内部函数。

使用标准函数时，函数名必须用大写字母；自变量必须用圆括号括起来；自变量必须是符合规定的BASIC表达式；三角函数中自变量的单位必须是弧度。下面再请看几个应用其他标准函数解决问题的例子。

# 1．符号函数

BASIC 语言中有一个符号函数 SGN(X),当自变量 $\mathbf { X } { > } 0$ 时，函数值为1、 $\mathbf { X } { = } 0$ 时函数值为0、 $\mathbf { X } { < } 0$ 时函数值为－1。

例7.4统计 $\mathbf { n }$ 个数据中正数、负数和零的个数。

分析：利用条件语句可以解决这个问题。利用符号函数也可以解决这个问题。请看下面的程序。

程序：

100 DIM A(2)   
105INPUT"数据个数:"；N   
110 FOR $\mathbf { I } = 1$ TO N   
120 READ X   
130 LET $\scriptstyle \mathbf { Y } = \mathbf { S } \mathbf { G } \mathbf { N } ( \mathbf { X } ) + 1$   
140 LET ${ \bf A ( Y ) } { = } { \bf A ( Y ) } { + } 1$   
150 NEXT I   
160 PRINT"正数有"；A(2)；"个"   
170 PRINT"负数有"；A(O)；"个"   
180 PRINT "零有"；A(1)；"个"   
190 END   
200 DATA · (从略)

程序中的语句5定义了一个含有三个下标变量的数组，用来统计三种数据的个数；语句 20把SGN(X的值加1后赋给变量Y,使得 $\mathbf X { > } 0$ 时 $\mathbf { Y }$ 的值为2、 $\mathbf { X } { = } 0$ 时 $\mathbf { Y }$ 的值为1、 $\mathbf { X } { < } \mathbf { 0 }$ 时 $\mathbf { Y }$ 的值为0;把Y的值当作下标变量的下标执行语句40时，恰好使A(0)统计负数的个数、A(1)统计0的个数、A(2)统计正数的个数（由于以前未给它们赋过值， ${ \bf A } ( 0 )$ 、A(1)、A(2)的初值为0）。在DATA语句中设置好 $\mathbf { n }$ 个数据后，运行这个程序就可以得到所需的结果。

# 2.对数函数

利用BASIC语言中的对数函数 $\mathbf { L O G } ( \mathbf { X } )$ 可以求X的自然对数，即以 $e$ 为底的X的对数。 $e$ 是一个无理数， $e { = } 2 . 7 1 8 2 8 \cdots$ 。实际应用中，经常要计算的是常用对数，所以应注意使用对数的换底公式。

例7.5设计一个程序，求N的 100次幂的整数有多少位 $( \mathbf { N } \neq \mathbf { \boldsymbol { 0 } } )$ 。

50 END

# 二、字符串函数

BASIC 语言有较强的处理字符串的功能，如检测字符串的长度、截取字符串中的某些字符、取字符串的ASCII代码值、根据ASCII代码值输出字符，等等。这些功能一般要通过调用字符

利用函数 $\mathrm { L E F T S } ( \mathbf { X } \mathbb { S } \ , \ n )$ 可以截取字符串 X\$中左起 $n$ 个字符组成的子字符串。其中 $n$ 的值必须不小于1,如果是小数，计算机会自动取整。计算机遇到调用LEFT\$函数的语句或命令时，会提供相应的子字符串。例如，运行上面的程序后，再键入

# 三、自定义函数

除了标准函数和字符串函数以外，BASIC语言还允许程序设计者自己定义函数。请看下面的程序。

10 DEF FN ${ \bf A } ( { \bf X } ) { = } { \bf X } { * } { \bf X } { * } { \bf X }$   
20 FOR $\mathbf { I } = 1$ TO 10  
30 PRINT FN A(①),  
40 LET $\mathbf { S } = \mathbf { S } + \mathbf { F } \mathbf { N }$ A(1)  
50 NEXT I  
60 PRINT  
70 PRINT $" \ : \mathbf { S } = "$ ; S  
80 END  
程序中的语句10叫做自定义函数语句。自定义函数语句的格式是

# DEF $<$ 函数名 $>$ （自变量） $= <$ 算术表达式 $>$

函数名必须以FN打头，后面跟一个可用于变量名的字符串，例如，在上面程序中的语句10里，用字母A与FN组成的函数名是FNA;圆括号中的自变量是一个形式参数，它只在自定义函数语句中起描述函数的作用，不会影响程序中变量的值，因此可以选用任何一个变量名；算术表达式的作用是告诉计算机如何进行计算，例如，语句10里用 $\mathbf { X } { * } \mathbf { X } { * } \mathbf { X }$ 告诉计算机要计算自变量的三次幂。

自定义函数语句的功能是定义一个函数，告诉计算机如何计算函数值，并给这个函数取一个名字。计算机执行自定义函数语句后，就可以像使用标准函数一样用函数名调用这个自定义函数了。

程序中的语句30调用自定义函数FNA,实际计算时用变量I的值代替形式参数X,然后显示FN A的值；语句40再次调用FNA,把函数值累加起来。运行这个程序，屏幕上会显示

输出了各项项数立方的一个数列以及从第一项到第十项的和。

使用自定义函数时应注意： $\textcircled{1}$ 自定义函数的自变量只能有一个； $\textcircled{2}$ 自定义函数的算术表达式中除了可以含有形式参数（自变量）以外，还可以含有程序中的其他变量和函数； $\textcircled{3}$ 自定义函数语句必须放在调用这个函数的语句之前； $\textcircled{4}$ 自定义函数可以重复定义，调用时只执行调用前最后一次定义时规定的计算。
