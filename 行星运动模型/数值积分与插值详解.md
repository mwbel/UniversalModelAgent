## 完整技术流程

### 1. JPL的工作（阶段1）

#### 1.1 数值积分

**输入**：
```python
# 牛顿运动方程 + 广义相对论修正
d²r/dt² = -GM/r² + Σ(摄动力)

# 初始条件（J2000.0时刻）
r₀ = [x₀, y₀, z₀]  # 位置
v₀ = [vx₀, vy₀, vz₀]  # 速度
```

**过程**：
```
1. 将1550-2650年分成3200个时间段（每段32天）
2. 对每个时间段：
   a) 使用RK积分器求解微分方程
   b) 每小时计算一次位置（32天×24小时 = 768个点）
   c) 得到该段的密集采样数据
3. 重复3200次
```

**计算量**：
- 3200段 × 768点 × 10天体 × 1000次迭代 ≈ 2.5×10¹⁰ 次微分方程求解
- 每次求解需要 ~10⁵ 次浮点运算
- 总计：~10¹⁵ 次浮点运算
- 耗时：数周（超级计算机）

#### 1.2 切比雪夫拟合

**目的**：数据压缩

**过程**：
```python
# 对每个时间段的768个采样点
# 拟合15个切比雪夫系数

# 最小二乘法
coeffs = solve_least_squares(
    T_matrix,  # 切比雪夫多项式矩阵
    positions  # 768个位置点
)

# 压缩比
原始数据：768个点 × 3坐标 = 2304个数字
压缩后：15个系数 × 3坐标 = 45个数字
压缩比：2304/45 ≈ 51:1
```

**输出**：de440.bsp文件
```
文件大小：114 MB
包含数据：
  - 3200个时间段
  - 10个天体
  - 每个天体每段：15系数 × 3坐标 = 45个数字
  - 总计：3200 × 10 × 45 ≈ 1.4M个数字
```

#### 2.3 切比雪夫插值（我的实现）

**核心算法**：

```python
# 1. 时间归一化到[-1, 1]
t_norm = 2 * (t - t_start) / (t_end - t_start) - 1

# 2. 计算切比雪夫多项式（递推）
T[0] = 1
T[1] = t_norm
for k in range(2, 15):
    T[k] = 2 * t_norm * T[k-1] - T[k-2]

# 3. 计算位置（点积）
x = Σ(coeffs_x[k] * T[k])  # k=0到14
y = Σ(coeffs_y[k] * T[k])
z = Σ(coeffs_z[k] * T[k])

# 4. 计算速度（导数）
dT[0] = 0
dT[1] = 1
for k in range(2, 15):
    dT[k] = 2 * T[k-1] + 2 * t_norm * dT[k-1] - dT[k-2]

vx = Σ(coeffs_x[k] * dT[k]) * dt_factor
vy = Σ(coeffs_y[k] * dT[k]) * dt_factor
vz = Σ(coeffs_z[k] * dT[k]) * dt_factor
```

**计算量**：
- 递推计算：15次乘法 + 15次加法
- 点积计算：15次乘法 + 15次加法
- 总计：~100次浮点运算
- 耗时：~0.067 毫秒

## 为什么这样设计？

### 设计权衡

| 方案 | 优点 | 缺点 | 适用性 |
|------|------|------|--------|
| **方案A：直接存储位置** | 查询最快 | 存储巨大（~100GB） | ✗ 不实用 |
| **方案B：实时数值积分** | 不需要存储 | 计算极慢（秒级） | ✗ 不实用 |
| **方案C：切比雪夫插值** | 存储小（114MB）<br>查询快（毫秒级） | 需要插值计算 | ✓ 最优方案 |

### 为什么选择切比雪夫多项式？

**对比其他插值方法**：

| 方法 | 精度 | 稳定性 | 计算速度 | 存储效率 |
|------|------|--------|----------|----------|
| 拉格朗日插值 | 高 | 差（振荡） | 快 | 中 |
| 样条插值 | 高 | 好 | 中 | 中 |
| **切比雪夫插值** | **极高** | **极好** | **快** | **高** |
| 傅里叶级数 | 高 | 好 | 快 | 中 |

**切比雪夫的优势**：
1. **最优逼近性**：在所有多项式中误差最小（Chebyshev定理）
2. **数值稳定**：不会出现Runge现象（高次振荡）
3. **递推高效**：O(n)时间计算所有多项式值
4. **压缩率高**：15个系数可表示768个点
