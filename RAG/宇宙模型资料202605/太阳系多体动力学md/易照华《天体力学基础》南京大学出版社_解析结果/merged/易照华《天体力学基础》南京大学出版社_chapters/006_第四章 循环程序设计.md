# 第四章 循环程序设计

应用顺序程序，让计算机按顺序执行一系列的语句，可以进行一些处理。但在实际应用中，会经常遇到需要反复进行一系列处理才能解决的问题，在这种情况下，本章要介绍的循环程序是一种非常得力的工具。

# 第一节循环结构的程序

我们知道，用BASIC语言的 SQR(X)函数可以输出一个数的算术平方根。要求出1到9这9个自然数的算术平方根，可以编出下面这段程序。

10 PRINT SQR(1)   
20 PRINT SQR(2)   
30 PRINT SQR(3)   
40 PRINT SQR(4)   
50 PRINT SQR(5)   
60 PRINT SQR(6)   
70 PRINT SQR(7)   
80 PRINT SQR(8)   
90 PRINT SQR(9)

照此办理，要想求出1到100这100个自然数的算术平方根，就得写出100条类似的语句来。类似地，在上一章的例3.11中，如果要统计输出全班40个学生的考试成绩，同样的程序要分别运行40次才行。这样做太麻烦了。BASIC语言提供了循环语句，可以使程序变得非常简明。请看下面的程序。

例4.1 输出自然数1到100的算术平方根的程序。

10 FOR $\mathbf { I } = 1$ TO 100 STEP 1  
20 PRINT SQR()  
30 NEXT I  
40 END  
计算机执行这个简短的程序时，屏幕上会显示出  
1  
1.41421356  
1.73205081  
2

… (从略) 9.89949495 9.94987438 10   
]

（显示时，屏幕上的内容会不断向上移动，按 $\mathbf { { C t r l + S } }$ 键可以使之暂停，再按任意一个字符键就可以继续显示)

这个程序里只有一个PRINT语句，为什么能输出 100个结果呢？好像计算机执行了100 次PRINT语句。实际上确实如此，这是因为10和30两个语句起了关键性的作用。

# 一、FOR-NEXT 语句

例4.1中的语句10叫做循环初始语句，也叫FOR语句。FOR语句中的变量I叫做循环控制变量，也叫循环变量；循环变量I后面的数1叫初值；TO后面的数100叫终值；STEP后面的数1叫步长值。语句30叫做循环终端语句，也叫NEXT语句。FOR语句的格式是

NEXT语句的格式是

# NEXT $<$ 循环变量名 $>$

在程序中，FOR语句和NEXT语句必须成对出现，二者使用的循环变量名必须相同。FOR语句中循环变量的初值、终值、步长值可以是数，也可以是变量，还可以是表达式。步长值是1时，FOR语句中的 STEP1可以省略不写。例如，例4.1中的语句10可以写成

10 FOR $\mathbf { I } = 1$ TO 100

计算机执行FOR语句时，首先把初值赋给循环变量，其次记下终值和步长值为以后的处理做好准备，然后开始执行FOR语句下面的语句。执行到NEXT语句时，计算机先让循环变量增加一个步长值，然后用增值后的循环变量值与记下的终值比较，如果没超过终值，就再次执行FOR 语句后面的语句，如果循环变量的值超过了终值，就执行NEXT语句后面的语句。

运行例4.1中的程序，计算机执行语句10时，首先让循环变量I变成初值1,然后记录下终值100 和步长值1就完成了语句10规定的操作;接下来计算机执行语句20,由于变量I的值已变成1,屏幕上显示出1的算术平方根；再接下来，计算机执行语句30。首先让循环变量I增加一个步长变成 $_ { \mathbf { I } + 1 }$ ,即变成2,然后把I的新值与记录下来的终值100比较，看看是否超过了终值，显然还没超过，于是就返回去第二次执行语句20;第二次执行语句20时，变量I的值变成了2,屏幕上显示出的是2的算术平方根；再次执行语句30时，先把变量I的值变成3,由于还没超过终值100,就再次返回去第三次执行语句20,，如此反复，直到第 100次执行语句30时，循环变量I的值变成101,超过了记录下来的终值100,这次计算机不再返回去执行语句20,而是向下执行这个NEXT语句下面的语句40。由于语句40是END语句，屏幕上输出了100的算术平方根后就结束了。

步长值不是1时，FOR 语句中必须给出 STEP和具体的步长值。例如，用

100 FOR $\mathbf { K } = \mathbf { 0 }$ TO 1000 STEP 3

120 PRINT K,

130 NEXT K

140 END

可以输出1000以内能被3整除的所有非负整数。

# 二、循环结构的程序

FOR 语句和NEXT语句好像一对“语句括号”，用它们把要重复执行的语句夹在中间就可以让计算机按规定的次数重复执行一系列的操作。夹在FOR 和NEXT语句之间的那部分语句构成循环体。程序运行过程中，循环变量的值控制着循环体的执行次数。由FOR 语句、循环体、NEXT语句组成的程序叫做循环结构的程序，简称循环程序。循环程序可以用图4.1所示的框图来描述。

从这个例子可以看到，循环变量的初值可以大于终值，这时步长小于0。此外，步长值、初值、终值也可以不是整数，但有时会因计算时的累计误差使循环次数达不到要求，因此，最好用整数来控制循环体的执行次数。

在上面的例题里，循环变量除了控制着循环体的执行次数以外，还参加了循环体的运算。实际应用时，循环变量可以不参加运算，只起控制循环体执行次数的作用。

# 二、数据的预置与读取

应用循环程序可以处理成批的数据。在此之前，提供数据的任务一般靠LET和INPUT语句来完成。要处理的数据很多时，使用LET和INPUT语句会很不方便。在这种情况下，可以使用DATA语句预先设置好数据，处理时再用READ语句把数据读入指定的变量中。请看下面的程序。

例4.6

10 READ A  
20 PRINT $" \mathbf { A } = "$ ；A  
30 READ B, C  
40 PRINT ${ \bf \mathrm { ~  ~ \Lambda ~ } } ^ { \prime \prime } \mathrm { \bf ~ B } = { \bf \mathrm { ~  ~ \Lambda ~ } } ; \mathrm { \bf ~ B } , ^ { \prime \prime } \mathrm { \bf ~ C } = { \bf \mathrm { ~  ~ \Lambda ~ } } ; \mathrm { \bf ~ C }$   
50 READ A,B, C  
60 PRINT $\mathbf { \ " A } = \mathbf { \ " }$ ；A, $\mathbf { \ " } \mathbf { B } = \mathbf { \ " }$ ； B, $" \mathbf { C } = "$ ; C  
70 DATA 11, 22,33,44,55,66,77  
80 END

如果不考虑10，30,50，70这几个语句，计算机执行这个程序会在标准位置上输出三行数值。由于没有LET或INPUT语句给变量A,B,C赋值，输出的数值应该是0。实际情况是否如此呢？不妨上机试一下。运行这个程序，屏幕上会显示出

$$
\mathsf { A } = 1 1
$$

B=22 C=33   
$\mathsf { A } = 4 4$ B=55 C=66   
]

显然，A,B,C这三个变量先后分别取得了11，22,33和44,55,66这6个数值。这6个数值恰好是语句80中列出的前6个数据的值。产生这种结果的原因是程序里使用了10,30,50和70语句。

语句 70叫做DATA语句，也叫数据语句或置数据语句。DATA语句的格式是

# DATA $<$ 数据表 $>$

其中的数据表是用逗号隔开的一组数据，数据表的末尾没有标点。BASIC程序输人后，如果程序里有DATA语句，计算机会设置一个数据指针，使这个指针指向程序中第一个DATA语句的第一个数据。

语句10,30,50叫做READ语句，也叫读数据语句。READ语句的格式是

# READ $<$ 变量名表 $>$

其中的变量名表可以是一个变量名，也可以是用逗号隔开的多个变量名，变量名表的末尾也不能有标点。

计算机执行READ语句时，从数据指针指出的那一个数据开始读取数据（变量名表中有几个变量，就读取几个数据)，并按照读取的顺序把数据值依次赋给变量表中的变量。与此同时，每读取一个数据，计算机就把数据指针移到DATA语句中的下一个数据上，为读取下一个数据指明位置。

DATA语句是非执行语句，计算机只是把其中的数据保存起来，为程序运行时读取这些数据的值作好准备。因此，DATA语句可以放在程序中任意位置上。为使程序清晰易读，一般把DATA语句放在END语句的前面，或者跟在相应的READ语句之后，也可以放在END语句的后面。DATA语句中的数据只能是常量，不能是变量或表达式。

数据较多，一个DATA语句放不下时，可以分成几个DATA语句。分开写时，所有数据的顺序要与写在一个DATA语句时一致。实际上，可以把程序中的多个DATA语句看作按行号顺序首尾相接的一个长的数据语句。在这种情况下，读完一个DATA语句中的数据后，计算机会把数据指针移到下一个DATA语句中的第一个数据上。

数据的个数不能少于程序中所有的READ语句要读取数据的次数，否则计算机会显示“OUTOFDATA ERROR $\mathbf { I N \ x x } ^ { \prime \prime }$ ，指出执行第xx语句时出现了“无数据可读”的错误。

例4.12某车站进站口通过一位乘客的时间是2到8秒，编程序估算进100位乘客约需多少时间。

10 LET $\mathbf { S } = 0$   
20 FOR $\mathbf { I } = 1$ TO 100  
30 LET $\mathbf { A } { = } \mathbf { R } \mathbf { N } \mathbf { D } ( 1 ) { * } 6 { + } 2$

50 NEXT I  
60 LET $\mathbf { S } = \mathbf { I N T } ( \mathbf { S } )$   
70 PRINT $\mathbf { \vec { S } } = \mathbf { \ " }$ ; S

# 本章小结

1.FOR-NEXT语句是构成循环程序的BASIC语句对，这一对语句控制着夹在它们之间的循环体的执行次数。设计循环程序时，应注意正确设置循环变量的初值、终值和步长值。

2．累加运算时，应在进入循环之前，为存放累加和的变量赋初始值0;累乘运算时，应在进入循环之前，为存放累乘积的变量赋初始值1。

3．用READ/DATA语句可以较方便地为变量提供批量数据。应注意使READ语句中的变量与DATA语句中相应的数据的类型保持一致。

4．取整函数 $\mathbf { I N I } ( \mathbf { X } )$ 的功能是取不大于自变量 $\mathbf { X }$ 的最大整数；随机数函数RND(X)的功能是产生一个大于0小于1的随机数。

5.利用公式 $\mathbf { R N D } ( 1 ) * ( \mathbf { M } - \mathbf { N } ) + \mathbf { N }$ 可以产生随机数R $( \mathbf { N } < \mathbf { R } < \mathbf { M } )$ ；利用公式 $\mathbf { I N T } ( \mathbf { R N D } ( 1 ) * ( \mathbf { M } - \mathbf { N } + 1 ) ) + \mathbf { N }$ 可以产生随机整数 $\mathbf { K } \left( \mathbf { N } < \mathbf { K } < \mathbf { M } \right)$ 。
