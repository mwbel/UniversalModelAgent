## 三、核心问题解答

### 3.1 JPL数值积分与插值算法的关系

**两阶段流程**：

```
阶段1：JPL的工作（离线，已完成）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
输入：物理定律 + 初始条件
  ↓
数值积分：求解微分方程（数周，超级计算机）
  ↓
切比雪夫拟合：数据压缩（1000:1）
  ↓
输出：de440.bsp（114MB）

阶段2：用户的工作（在线，实时）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
输入：de440.bsp + 查询时间
  ↓
切比雪夫插值：多项式求值（0.067ms）
  ↓
输出：位置和速度
```

**关键点**：
- 阶段1已完成，不需要重做
- 阶段2是我的实现，用于实时查询
- 两者分工明确，各司其职

### 3.2 为什么不需要数值积分

**答案**：JPL已经完成了数值积分，结果存储在de440.bsp中。

**类比**：
- 数值积分 = 拍摄电影（耗时，一次性）
- 切比雪夫拟合 = 视频压缩（H.264）
- de440.bsp = MP4文件
- 插值计算 = 视频播放（实时）

**你不需要重新拍摄电影，只需要播放MP4文件！**

2. **插值精度**：圆轨道误差 < 1e-4 ✓
